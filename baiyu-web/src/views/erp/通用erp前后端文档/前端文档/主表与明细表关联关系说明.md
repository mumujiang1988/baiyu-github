# 主表与明细表关联关系说明

## 一、数据关联核心机制

### 1.1 关联字段

主表与明细表通过**单据编号(billNo)**或**主表ID**进行关联:

```
主表(t_sale_order)
├── id (主键)
├── FBillNo (单据编号) ← 关联字段
├── FDate
└── FAmount

明细表(t_sale_order_entry)
├── id (主键)
├── order_id (外键) ← 关联字段,存储主表的FBillNo或id
├── fMaterialId
└── fQty
```

### 1.2 关联方式

有两种常见的关联方式:

#### 方式一:通过单据编号关联(推荐)

```sql
-- 主表
SELECT * FROM t_sale_order WHERE FBillNo = 'SO2024010001'

-- 明细表
SELECT * FROM t_sale_order_entry WHERE order_id = 'SO2024010001'
```

**优点:**
- 单据编号具有业务含义,便于理解和调试
- 单据编号唯一,不会因ID变化而失效
- 适合跨系统数据交互

#### 方式二:通过主表ID关联

```sql
-- 主表
SELECT * FROM t_sale_order WHERE id = 123

-- 明细表
SELECT * FROM t_sale_order_entry WHERE order_id = 123
```

**优点:**
- 查询性能更好(数字比较)
- 外键约束更严格
- 适合单系统内部关联

### 3.2 加载表单数据流程

```javascript
async function loadFormData(id) {
  // 步骤1: 加载主表数据
  const apiMethod = await getApiMethod('get')
  const response = await apiMethod(id)
  formData.value = response.data || response
  
  // 步骤2: 获取单据编号
  const billNoField = parsedConfig.page?.billNoField || 'FBillNo'
  const billNoValue = formData.value[billNoField] || formData.value.FBillNo
  
  // 步骤3: 加载明细数据
  if (billNoValue) {
    // 3.1 加载明细表数据
    const entryMethod = await getApiMethod('entry')
    if (entryMethod) {
      const entryResponse = await entryMethod(billNoValue)
      const entryData = getResponseResult(entryResponse, [])
      entryList.value = Array.isArray(entryData) ? entryData : []
    }
    
    // 3.2 加载成本表数据
    const costMethod = await getApiMethod('cost')
    if (costMethod) {
      const costResponse = await costMethod(billNoValue)
      const costResult = getResponseResult(costResponse, {})
      costData.value = typeof costResult === 'object' ? costResult : {}
    }
  }
}
```

  3. 删除成本表记录: DELETE FROM t_sale_order_cost WHERE order_id = FBillNo

    ↓

- 显示成功提示
- 刷新主表列表

### 4.2 级联删除策略

#### 策略一:后端级联删除(推荐)

```sql
-- 在数据库层面设置外键级联删除
ALTER TABLE t_sale_order_entry 
ADD CONSTRAINT fk_order_entry 
FOREIGN KEY (order_id) REFERENCES t_sale_order(FBillNo)
ON DELETE CASCADE;

-- 删除主表记录时,自动删除关联的明细记录
DELETE FROM t_sale_order WHERE id = 1;
-- 数据库自动执行:
-- DELETE FROM t_sale_order_entry WHERE order_id = 'SO2024010001';
```

**优点:**
- 数据一致性有保障
- 代码简洁
- 性能较好

#### 策略二:应用层手动删除

```java
// 后端Service层代码
public void deleteOrder(List<Long> ids) {
    for (Long id : ids) {
        // 1. 查询主表记录
        SaleOrder order = orderMapper.selectById(id);
        
        // 2. 删除明细记录
        entryMapper.deleteByOrderId(order.getFBillNo());
        
        // 3. 删除成本记录
        costMapper.deleteByOrderId(order.getFBillNo());
        
        // 4. 删除主表记录
        orderMapper.deleteById(id);
    }
}
```

**优点:**
- 逻辑清晰可控
- 可以添加业务校验
- 可以记录删除日志

## 七、关键代码实现

### 7.1 主表查询

```javascript
const getList = async () => {
  loading.value = true
  try {
    const mainQueryConfig = buildMainQueryConfig()
    const tableName = getTableNameFromConfig()
    const moduleCode = currentConfig.value?.moduleCode
    
    const response = await request({
      url: '/erp/engine/query/execute',
      method: 'post',
      data: {
        moduleCode,
        tableName,
        queryConfig: mainQueryConfig,
        pageNum: queryParams.value.pageNum,
        pageSize: queryParams.value.pageSize
      }
    })
    
    tableData.value = response.data?.rows || []
    total.value = response.data?.total || 0
  } catch (error) {
    ElMessage.error('查询列表失败')
  } finally {
    loading.value = false
  }
}
```

### 7.2 明细表查询

```javascript
const loadSubTablesByBillNo = async (billNo) => {
  try {
    const moduleCode = currentConfig.value?.moduleCode
    const subTableConfigs = multiTableQueryBuilder.parseSubTableConfigs(currentConfig.value)
    
    // 并行查询所有子表格
    const results = await multiTableQueryBuilder.queryAllSubTables(
      moduleCode,
      subTableConfigs,
      { billNo }  // 上下文数据,用于替换 ${billNo}
    )
    
    // 存储明细数据
    if (results.entry) {
      entryList.value = results.entry.data
    }
    
    // 存储成本数据
    if (results.cost) {
      costData.value = results.cost.data[0] || {}
    }
  } catch (error) {
    console.error('查询子表格失败:', error)
  }
}
```

### 7.3 修改提交

```javascript
const handleSubmit = async () => {
  try {
    await formRef.value.validate()
    submitLoading.value = true
    
    // 合并主表和明细表数据
    const submitData = { ...formData.value }
    if (entryList.value && entryList.value.length > 0) {
      submitData.entryList = entryList.value
    }
    if (costData.value && Object.keys(costData.value).length > 0) {
      submitData.costData = costData.value
    }
    
    // 判断新增或修改
    const isNew = !submitData.id
    const apiMethod = await getApiMethod(isNew ? 'add' : 'update')
    
    await apiMethod(submitData)
    ElMessage.success(isNew ? '新增成功' : '修改成功')
    
    dialogVisible.value = false
    getList()
  } catch (error) {
    ElMessage.error('保存失败：' + (error.message || '请检查表单填写是否正确'))
  } finally {
    submitLoading.value = false
  }
}
```

## 八、最佳实践建议

### 8.1 关联字段选择

**推荐使用单据编号作为关联字段:**

```javascript
// 配置
{
  "pageConfig": {
    "billNoField": "FBillNo"  // 单据编号字段
  },
  "detailConfig": {
    "detail": {
      "tabs": [{
        "queryConfig": {
          "defaultConditions": [{
            "field": "order_id",
            "operator": "eq",
            "value": "${billNo}"  // 使用单据编号关联
          }]
        }
      }]
    }
  }
}
```

**原因:**
1. 单据编号具有业务含义,便于理解和调试
2. 单据编号唯一且稳定,不会因数据迁移而改变
3. 适合跨系统数据交互和报表统计

### 8.2 数据一致性保障

**方式一:数据库外键约束**

```sql
-- 创建外键约束
ALTER TABLE t_sale_order_entry
ADD CONSTRAINT fk_order_entry
FOREIGN KEY (order_id) REFERENCES t_sale_order(FBillNo)
ON DELETE CASCADE;  -- 级联删除

-- 创建索引提高查询性能
CREATE INDEX idx_order_id ON t_sale_order_entry(order_id);
```

**方式二:应用层事务控制**

```java
@Transactional
public void updateOrder(OrderDTO orderDTO) {
    // 1. 更新主表
    orderMapper.updateById(orderDTO);
    
    // 2. 删除旧明细
    entryMapper.deleteByOrderId(orderDTO.getFBillNo());
    
    // 3. 插入新明细
    for (Entry entry : orderDTO.getEntryList()) {
        entry.setOrderId(orderDTO.getFBillNo());  // 设置关联字段
        entryMapper.insert(entry);
    }
}
```

### 8.3 性能优化建议
