### 2.2 查询条件操作符说明

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `eq` | 等于 | `{ "field": "status", "operator": "eq", "value": "1" }` |
| `ne` | 不等于 | `{ "field": "status", "operator": "ne", "value": "0" }` |
| `like` | 模糊匹配 | `{ "field": "name", "operator": "like", "value": "张" }` |
| `in` | 包含于 | `{ "field": "type", "operator": "in", "value": ["A", "B"] }` |
| `between` | 区间 | `{ "field": "date", "operator": "between", "value": ["2024-01-01", "2024-12-31"] }` |
| `gt` | 大于 | `{ "field": "amount", "operator": "gt", "value": 1000 }` |
| `gte` | 大于等于 | `{ "field": "amount", "operator": "gte", "value": 1000 }` |
| `lt` | 小于 | `{ "field": "amount", "operator": "lt", "value": 5000 }` |
| `lte` | 小于等于 | `{ "field": "amount", "operator": "lte", "value": 5000 }` |

### 2.3 模板变量说明

在查询条件中可以使用模板变量,系统会自动替换:

| 变量 | 说明 | 来源 |
|------|------|------|
| `${FBillNo}` | 单据编号 | 从主表行数据获取(使用主表字段名) |
| `${FCustId}` | 客户ID | 从主表行数据获取 |
| `${userId}` | 当前用户ID | 从登录信息获取 |
| `${deptId}` | 当前部门ID | 从登录信息获取 |

**示例:**
```javascript
{
  "field": "fbillno",
  "operator": "eq",
  "value": "${FBillNo}"  // 会被替换为实际的单据编号
}
```

**重要说明:**
- 模板变量使用主表的实际字段名(如`${FBillNo}`)
- 关联字段使用明细表的字段名(如`fbillno`)
- 确保主表字段名与明细表关联字段的数据一致

### 3.3 新增按钮事件流程

```
用户点击"新增"按钮
    ↓
调用 openDialog('add')
    ↓
初始化 formData 为默认值
    ↓
清空 entryList 和 costData
    ↓
设置 dialogTitle = "新增XXX"
    ↓
显示对话框
    ↓
用户填写表单
    ↓
点击"确定"按钮
    ↓
调用 handleSubmit()
    ↓
验证表单规则
    ↓
合并 formData + entryList + costData
    ↓
调用 getApiMethod('add')
    ↓
发送新增请求
    ↓
成功后关闭对话框并刷新列表
```

### 3.4 修改按钮事件流程

```
用户选择一条数据
    ↓
点击"修改"按钮
    ↓
调用 openDialog('edit')
    ↓
检查是否只选择了一条数据
    ↓
调用 loadFormData(id)
    ↓
调用 getApiMethod('get') 获取主表数据
    ↓
调用 getApiMethod('entry') 获取明细数据
    ↓
调用 getApiMethod('cost') 获取成本数据
    ↓
填充 formData、entryList、costData
    ↓
显示对话框
    ↓
用户修改数据
    ↓
点击"确定"按钮
    ↓
调用 handleSubmit()
    ↓
验证表单规则
    ↓
合并 formData + entryList + costData
    ↓
调用 getApiMethod('update')
    ↓
发送修改请求
    ↓
成功后关闭对话框并刷新列表
```

    ↓

## 六、API配置

### 5.1 API方法配置

```javascript
{
  "apiConfig": {
    "methods": {
      // 字符串格式(简单配置)
      "get": "/api/saleorder/get",
      "add": "/api/saleorder/add",
      "update": "/api/saleorder/update",
      "delete": "/api/saleorder/delete",
      "entry": "/api/saleorder/entry",
      "cost": "/api/saleorder/cost",
      
      // 对象格式(详细配置)
      "audit": {
        "url": "/api/saleorder/audit",
        "method": "post"
      },
      "unAudit": {
        "url": "/api/saleorder/unAudit",
        "method": "post"
      }
    }
  }
}
```

### 5.2 API方法说明

| 方法名 | 说明 | 请求方式 | 参数 |
|--------|------|----------|------|
| `get` | 获取单条数据 | GET | `{ id: xxx }` |
| `add` | 新增数据 | POST | formData |
| `update` | 修改数据 | POST | formData |
| `delete` | 删除数据 | POST | `{ ids: [...] }` |
| `entry` | 获取明细数据 | GET | `{ billNo: xxx }` |
| `cost` | 获取成本数据 | GET | `{ billNo: xxx }` |
| `audit` | 审核 | POST | `{ ids: [...] }` |
| `unAudit` | 反审核 | POST | `{ ids: [...] }` |

## 七、后端接口规范

### 7.1 通用查询接口

**接口地址:** `POST /erp/engine/query/execute`

**请求参数:**
```json
{
  "moduleCode": "saleorder",
  "tableName": "t_sale_order",
  "queryConfig": {
    "conditions": [
      {
        "field": "FStatus",
        "operator": "eq",
        "value": "1"
      }
    ],
    "orderBy": [
      {
        "field": "FCreateDate",
        "direction": "DESC"
      }
    ]
  },
  "pageNum": 1,
  "pageSize": 10
}
```

**响应格式:**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "rows": [
      {
        "id": 1,
        "FBillNo": "SO2024010001",
        "FDate": "2024-01-15",

  }
}
```

## 九、常见问题

### 9.1 字典数据不显示

**原因:** 字典配置错误或字典接口未返回数据

**解决:**
1. 检查 `dictionaryConfig` 配置是否正确
2. 检查字典接口是否正常返回数据
3. 检查字段名是否匹配

### 9.2 明细表格数据不加载

**原因:** 查询条件配置错误或表名不正确

**解决:**
1. 检查 `detailConfig.detail.tabs` 配置
2. 检查 `tableName` 是否正确
3. 检查查询条件中的字段名是否匹配
4. 检查模板变量是否正确替换

### 9.3 新增/修改提交失败

**原因:** API配置错误或数据格式不正确

**解决:**
1. 检查 `apiConfig.methods` 配置
2. 检查后端接口是否正常
3. 检查提交的数据格式是否符合后端要求

## 十、最佳实践

### 10.1 配置优化建议

1. **合理使用页签:** 不要在一个抽屉中放置过多页签,建议不超过5个
2. **优化查询条件:** 只查询必要的字段,避免查询大量无用数据
3. **使用模板变量:** 提高配置的通用性和可维护性
4. **配置权限控制:** 为每个操作按钮配置权限,确保安全性

### 10.2 性能优化建议

1. **分页查询:** 主表使用分页,避免一次查询过多数据
2. **懒加载:** 明细数据在需要时才加载
3. **缓存字典:** 字典数据使用缓存,减少重复请求
4. **并行查询:** 多个子表格使用并行查询,提高加载速度
