# ERP 字典接口优化实施文档

## 📋 更新概览

### 后端接口变更

**新接口：**
- `GET /erp/engine/dict/union/{dictType}` - 分段返回字典数据
- `GET /erp/engine/dict/all` - 查询所有字典数据

**返回格式：**
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "dictTypeList": [
      {
        "label": "用户状态",
        "value": "user_status",
        "type": "user_status"
      }
    ],
    "dictDataList": [
      {
        "label": "正常",
        "value": "0",
        "type": "user_status"
      },
      {
        "label": "禁用",
        "value": "1",
        "type": "user_status"
      },
      {
        "label": "原材料 A",
        "value": "YW001",
        "type": "product_category"
      }
    ]
  }
}
```

**数据结构说明：**
- `dictTypeList`: sys_dict_type（字典类型表）→ 第一段
- `dictDataList`: sys_dict_data + bymaterial_dictionary（系统字典 + 业务字典合并）→ 第二段

## 🎯 前端优化方案

### 1. DictionaryBuilder.js 优化

**文件路径：** `src/views/erp/utils/DictionaryBuilder.js`

#### 主要变更：

##### 1.1 buildFromConfig 方法
```javascript
buildFromConfig(dictionaryConfig, moduleCode) {
  const dicts = dictionaryConfig.dicts || []
  
  for (const dict of dicts) {
    const { dictKey, dictType, data, tableName, searchable } = dict
    
    if (Array.isArray(data)) {
      // 静态字典
      this.register(dictKey, DictionaryBuilder.buildStatic(dictKey, data))
    } else if (tableName) {
      // ✅ 动态字典 - 使用新的统一字典接口
      const config = {
        ...dict,
        type: searchable ? 'remote' : 'dynamic',
        api: `/erp/engine/dict/union/${dictType}`,  // ← 新接口路径
        searchApi: searchable ? `/erp/engine/dictionary/search/${dictKey}?moduleCode=${moduleCode}&keyword={keyword}` : undefined
      }
      this.register(dictKey, DictionaryBuilder.buildDynamic(dictKey, config))
    }
  }
  
  return this
}
```

##### 1.2 preloadAll 方法
```javascript
async preloadAll(moduleCode) {
  const promises = []

  this.registry.forEach((builder, name) => {
    if (builder.type === 'dynamic') {
      const promise = builder.load(async () => {
        // ✅ 调用新接口
        const response = await request({
          url: `/erp/engine/dict/union/${name}`,
          method: 'get'
        })
        
        let data = []
        if (response.code === 200 || response.errorCode === 0) {
          // ✅ 处理分段返回的数据
          const result = response.data || response
          // 合并两段数据
          data = [
            ...(result.dictTypeList || []),
            ...(result.dictDataList || [])
          ]
        } else if (Array.isArray(response)) {
          data = response
        }
        
        console.log(`字典加载成功：${name}, 共 ${data.length} 条`)
        return data
      })
      promises.push(promise)
    }
  })

  await Promise.all(promises)
  console.log('字典预加载完成')
}
```

### 2. DictionaryLoader.js 新增工具类

**文件路径：** `src/views/erp/utils/DictionaryLoader.js`

#### 核心功能：

##### 2.1 加载单个字典
```javascript
import DictionaryLoader from '@/views/erp/utils/DictionaryLoader'

// 加载单个字典（按类型）
const data = await DictionaryLoader.loadByType('currency')
console.log('字典类型:', data.dictTypeList)
console.log('字典数据:', data.dictDataList)
console.log('合并数据:', data.allData)
```

##### 2.2 加载所有字典
```javascript
// 加载所有字典
const allData = await DictionaryLoader.loadAll()
console.log('所有字典类型:', allData.dictTypeList)
console.log('所有字典数据:', allData.dictDataList)
```

##### 2.3 仅加载字典类型
```javascript
// 仅加载字典类型列表
const types = await DictionaryLoader.loadDictTypes()
console.log('字典类型列表:', types)

// 按类型过滤
const currencyTypes = await DictionaryLoader.loadDictTypes('currency')
```

##### 2.4 仅加载字典数据
```javascript
// 仅加载字典数据列表
const dataList = await DictionaryLoader.loadDictData('currency')
console.log('币别字典数据:', dataList)
```

##### 2.5 批量加载
```javascript
// 批量加载多个字典类型
const dictTypes = ['currency', 'payment_clause', 'product_category']
const resultMap = await DictionaryLoader.loadBatch(dictTypes)

// 获取特定类型的字典
const currencyData = resultMap.get('currency')
console.log('币别字典:', currencyData)
```

##### 2.6 带缓存的加载
```javascript
// 加载字典（带缓存，默认 5 分钟 TTL）
const cachedData = await DictionaryLoader.loadWithCache('currency')

// 自定义缓存时间
const customCacheData = await DictionaryLoader.loadWithCache('currency', 10 * 60 * 1000)

// 清除缓存
DictionaryLoader.clearCache('currency')  // 清除单个
DictionaryLoader.clearCache()            // 清除所有
```

## 💡 使用场景

### 场景 1：页面初始化加载字典

```vue
<script setup>
import { onMounted } from 'vue'
import DictionaryLoader from '@/views/erp/utils/DictionaryLoader'
import engine from '@/views/erp/utils/DictionaryBuilder'

onMounted(async () => {
  // 方式 1：使用 DictionaryBuilder 预加载
  await engine.preloadAll('sale_order')
  
  // 方式 2：使用 DictionaryLoader 批量加载
  const dictTypes = ['currency', 'payment_clause', 'trade_way']
  const dataMap = await DictionaryLoader.loadBatch(dictTypes)
  
  // 注册到构建器
  dataMap.forEach((data, type) => {
    engine.register(type, DictionaryBuilder.buildDynamic(type, {
      type: 'dynamic',
      ttl: 5 * 60 * 1000
    }))
  })
})
</script>
```

### 场景 2：下拉框选择器

```vue
<template>
  <el-select v-model="form.currency" placeholder="请选择币别">
    <el-option
      v-for="item in currencyOptions"
      :key="item.value"
      :label="item.label"
      :value="item.value"
    />
  </el-select>
</template>

<script setup>
import { ref, onMounted } from 'vue'
import DictionaryLoader from '@/views/erp/utils/DictionaryLoader'

const currencyOptions = ref([])

onMounted(async () => {
  // 加载币别字典
  const data = await DictionaryLoader.loadWithCache('currency')
  currencyOptions.value = data.dictDataList
})
</script>
```

### 场景 3：表格列字典渲染

```vue
<template>
  <el-table-column 
    prop="currency" 
    label="币别"
    :formatter="formatCurrency"
  />
</template>

<script setup>
import { DictionaryBuilderEngine } from '@/views/erp/utils/DictionaryBuilder'

const engine = new DictionaryBuilderEngine()

// 格式化函数
const formatCurrency = (row) => {
  const options = engine.getData('currency')
  const item = options.find(opt => opt.value === row.currency)
  return item?.label || row.currency
}
</script>
```

## 📊 性能优化建议

### 1. 缓存策略
```javascript
// 推荐：使用缓存加载
const data = await DictionaryLoader.loadWithCache('currency', 5 * 60 * 1000)

// 不推荐：每次都重新加载
const data = await DictionaryLoader.loadByType('currency')
```

### 2. 批量加载
```javascript
// 推荐：批量加载，减少请求次数
const dictTypes = ['currency', 'payment_clause', 'trade_way']
const dataMap = await DictionaryLoader.loadBatch(dictTypes)

// 不推荐：逐个加载
const currency = await DictionaryLoader.loadByType('currency')
const payment = await DictionaryLoader.loadByType('payment_clause')
const trade = await DictionaryLoader.loadByType('trade_way')
```

### 3. 按需加载
```javascript
// 推荐：只加载需要的部分
const types = await DictionaryLoader.loadDictTypes()  // 仅类型
const dataList = await DictionaryLoader.loadDictData('currency')  // 仅数据

// 不推荐：加载全部然后过滤
const all = await DictionaryLoader.loadByType('currency')
```

## 📝 总结

### 优势

1. **统一接口** - 一个接口返回所有字典数据
2. **分段结构** - 清晰区分字典类型和字典数据
3. **灵活使用** - 可以单独获取类型或数据
4. **性能优化** - 支持缓存、批量加载
5. **向后兼容** - 配置格式无需修改

### 关键变更点

- ✅ `DictionaryBuilder.buildFromConfig()` - API 路径更新
- ✅ `DictionaryBuilder.preloadAll()` - 数据处理逻辑更新
- ✅ 新增 `DictionaryLoader` 工具类
- ✅ 后端接口改为 `/erp/engine/dict/union/{dictType}`

### 推荐实践

1. 优先使用 `DictionaryLoader.loadWithCache()` 带缓存的加载
2. 多个字典使用 `loadBatch()` 批量加载
3. 仅需类型或数据时使用对应的专用方法
4. 页面卸载时考虑清理缓存（可选）
