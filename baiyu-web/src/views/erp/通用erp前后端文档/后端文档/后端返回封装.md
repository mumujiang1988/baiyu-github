# ERP 低代码模块标准返回类 - ErpResponse

## 📋 概述

`ErpResponse` 是 ERP 低代码模块独立的标准响应封装类，**不依赖 RuoYi 框架的 R 类**。

### 核心特性

- ✅ **独立于 RuoYi 框架**：无需依赖 `com.ruoyi.common.core.domain.R`
- ✅ **完全兼容 HTTP 状态码**：200(成功)、500(失败)、300(警告)
- ✅ **丰富的构造方法**：支持多种场景的响应构建
- ✅ **流式 API 支持**：支持链式调用设置额外字段
- ✅ **时间戳支持**：自动记录响应时间戳
- ✅ **类型安全**：完整的泛型支持

---

## 🏗️ 类结构

```java
package com.ruoyi.erp.domain.response;

public class ErpResponse<T> {
    private int code;              // 状态码
    private String msg;            // 返回消息
    private String detailMessage;  // 详细错误信息（可选）
    private T data;                // 返回数据
    private long timestamp;        // 时间戳
}
```

## 💡 使用示例

### 1. Controller 中的基本用法

#### 成功响应

```java
@GetMapping("/list")
public ErpResponse<List<User>> list() {
    List<User> users = userService.list();
    return ErpResponse.ok(users);
}
```

#### 带消息的成功响应

```java
@PostMapping("/add")
public ErpResponse<Void> add(@RequestBody User user) {
    int result = userService.add(user);
    return result > 0 ? ErpResponse.ok("添加成功") : ErpResponse.fail("添加失败");
}
```

#### 带数据和消息的成功响应

```java
@GetMapping("/{id}")
public ErpResponse<User> getById(@PathVariable Long id) {
    User user = userService.getById(id);
    return ErpResponse.ok("查询成功", user);
}
```

#### 失败响应

```java
@DeleteMapping("/{id}")
public ErpResponse<Void> delete(@PathVariable Long id) {
    try {
        userService.delete(id);
        return ErpResponse.ok();
    } catch (Exception e) {
        return ErpResponse.fail("删除失败：" + e.getMessage());
    }
}
```

#### 带详细错误信息的失败响应

```java
@PostMapping("/import")
public ErpResponse<Void> upload(@RequestParam MultipartFile file) {
    try {
        if (file.isEmpty()) {
            return ErpResponse.fail("上传文件不能为空");
        }
        importService.importData(file);
        return ErpResponse.ok("导入成功");
    } catch (Exception e) {
        return ErpResponse.fail(500, "导入失败", e.getMessage());
    }
}
```

### 2. 静态方法速查表

| 方法 | 说明 | 示例 |
|------|------|------|
| `ok()` | 无数据成功响应 | `ErpResponse.ok()` |
| `ok(T data)` | 带数据的成功响应 | `ErpResponse.ok(userList)` |
| `ok(String msg)` | 带消息的成功响应 | `ErpResponse.ok("操作成功")` |
| `ok(String msg, T data)` | 带消息和数据的成功响应 | `ErpResponse.ok("查询成功", user)` |
| `fail()` | 无消息的失败响应 | `ErpResponse.fail()` |
| `fail(String msg)` | 带消息的失败响应 | `ErpResponse.fail("操作失败")` |
| `fail(String msg, T data)` | 带消息和数据的失败响应 | `ErpResponse.fail("失败", errorInfo)` |
| `fail(int code, String msg)` | 指定状态码的失败响应 | `ErpResponse.fail(404, "资源不存在")` |
| `fail(int code, String msg, String detailMessage)` | 带详细错误的失败响应 | `ErpResponse.fail(500, "系统错误", e.getMessage())` |
| `warn(String msg)` | 警告响应 | `ErpResponse.warn("数据已存在")` |
| `warn(String msg, T data)` | 带数据的警告响应 | `ErpResponse.warn("缓存未命中", cacheKey)` |

### 3. 工具方法

#### 判断响应状态

```java
ErpResponse<User> response = userService.getUser(id);

if (response.isSuccess()) {
    System.out.println("查询成功");
}

if (response.isError()) {
    System.out.println("查询失败：" + response.getMsg());
}

if (response.isWarn()) {
    System.out.println("警告信息：" + response.getMsg());
}
```

#### 链式调用

```java
return ErpResponse.fail(500, "系统异常")
    .withDetailMessage(e.getClass().getName())
    .withTimestamp(System.currentTimeMillis());
```

## 🔄 与 RuoYi R 类的区别

| 特性 | ErpResponse | RuoYi R |
|------|------------|---------|
| 包路径 | `com.ruoyi.erp.domain.response` | `com.ruoyi.common.core.domain` |
| 框架依赖 | 独立，无外部依赖 | 依赖 RuoYi 框架 |
| 状态码常量 | SUCCESS=200, FAIL=500, WARN=300 | SUCCESS=200, FAIL=500 |
| 时间戳 | ✅ 支持 | ❌ 不支持 |
| 链式调用 | ✅ 支持 | ❌ 不支持 |
| 适用场景 | ERP 低代码模块专用 | RuoYi 全局通用 |

## 📦 完整代码示例

### ErpPageConfigController 示例

```java
package com.ruoyi.erp.controller.erp;

import com.ruoyi.erp.domain.response.ErpResponse;
import com.ruoyi.erp.service.ErpPageConfigService;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.web.bind.annotation.*;

@Slf4j
@RestController
@RequestMapping("/erp/config")
@RequiredArgsConstructor
public class ErpPageConfigController {

    private final ErpPageConfigService pageConfigService;

    /**
     * 获取配置详情
     */
    @GetMapping("/{configId}")
    public ErpResponse<ErpPageConfigVo> getInfo(@PathVariable Long configId) {
        return ErpResponse.ok(pageConfigService.selectById(configId));
    }

    /**
     * 新增配置
     */
    @PostMapping()
    public ErpResponse<Void> add(@RequestBody ErpPageConfigBo bo) {
        return pageConfigService.insertByBo(bo) > 0 
            ? ErpResponse.ok("添加成功") 
            : ErpResponse.fail("添加失败");
    }

    /**
     * 删除配置
     */
    @DeleteMapping("/{configIds}")
    public ErpResponse<Void> remove(@PathVariable Long[] configIds) {
        return pageConfigService.deleteByIds(configIds) > 0 
            ? ErpResponse.ok("删除成功") 
            : ErpResponse.fail("删除失败");
    }

    /**
     * 获取页面配置
     */
    @GetMapping("/get/{moduleCode}")
    public ErpResponse<String> getPageConfig(@PathVariable String moduleCode) {
        String config = pageConfigService.getPageConfig(moduleCode);
        if (config == null) {
            return ErpResponse.fail("未找到配置");
        }
        return ErpResponse.ok(moduleCode, config);
    }
}
```

## 🎯 最佳实践

### 1. 统一返回风格

```java
// ✅ 推荐：统一使用 ErpResponse
@GetMapping("/list")
public ErpResponse<List<User>> list() {
    return ErpResponse.ok(userService.list());
}

// ❌ 不推荐：混用 R 和 ErpResponse
@GetMapping("/list")
public R<List<User>> list() {
    return R.ok(userService.list());
}
```

### 2. 错误处理

```java
// ✅ 推荐：详细的错误信息
@PostMapping("/save")
public ErpResponse<Void> save(@RequestBody Data data) {
    try {
        service.save(data);
        return ErpResponse.ok("保存成功");
    } catch (ValidationException e) {
        return ErpResponse.fail(400, "数据验证失败：" + e.getMessage());
    } catch (Exception e) {
        log.error("保存失败", e);
        return ErpResponse.fail(500, "系统异常", e.getMessage());
    }
}
```

### 3. 空值处理

```java
// ✅ 推荐：明确的空值处理
@GetMapping("/{id}")
public ErpResponse<User> getById(@PathVariable Long id) {
    User user = service.getById(id);
    return user != null 
        ? ErpResponse.ok(user) 
        : ErpResponse.fail("用户不存在");
}
```

## 🔧 迁移指南

### 从 RuoYi R 类迁移到 ErpResponse

#### Step 1: 修改导入语句

```java
// 旧代码
import com.ruoyi.common.core.domain.R;

// 新代码
import com.ruoyi.erp.domain.response.ErpResponse;
```

#### Step 2: 替换返回类型

```java
// 旧代码
public R<User> getUser() {
    return R.ok(user);
}

// 新代码
public ErpResponse<User> getUser() {
    return ErpResponse.ok(user);
}
```

#### Step 3: 替换静态方法

```java
// 旧代码
return R.ok(data);
return R.fail("错误");

// 新代码
return ErpResponse.ok(data);
return ErpResponse.fail("错误");
```

## 📝 注意事项

1. **不要混用**：同一模块内不要混用 `R` 和 `ErpResponse`
2. **前端适配**：确保前端能正确处理新的响应格式
3. **异常处理**：配合全局异常处理器使用效果更佳
4. **日志记录**：建议在关键位置记录响应日志
