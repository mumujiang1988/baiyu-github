## 一、审计概述

审计时间: 2024年

审计工具: 代码审查、静态分析

## 二、发现的问题

### 2.1 高优先级问题 (安全隐患)

#### 2.1.1 SQL 注入风险
**文件**: `DynamicQueryEngine.java`, `VirtualFieldService.java`

**问题描述**:
- `DynamicQueryEngine.buildQueryConditions()` 方法直接使用配置中的字段名构建查询条件,未做安全校验
- `VirtualFieldService.buildBatchQuerySql()` 方法直接拼接表名和字段名到 SQL 语句

**代码位置**:
- `DynamicQueryEngine.java:58` 第58行注释提到"简单校验字段非空即可",但实际未做安全校验
- `VirtualFieldService.java:165-178`, SQL 拼接逻辑

**影响范围**: 
- 可能导致 SQL 注入攻击
- 数据泄露风险

**修复建议**:
```java
// 1. 增加字段白名单校验
private static final Set<String> ALLOWED_FIELDS = Set.of(
    "field1", "field2", "field3" "status", "create_time", "update_time"
);

private void validateField(String field) {
    if (!ALLOWED_FIELDS.contains(field)) {
        throw new IllegalArgumentException("非法字段名: " + field);
    }
}

// 2. 使用参数化查询
String sql = "SELECT :sourceField, :displayField FROM :tableName WHERE :sourceField IN (:values)";
MapSqlParameterSource params = new MapSqlParameterSource();
params.addValue("sourceField", sourceField);
```

**优先级**: 🔴 高
**工作量**: 2-3 天

### 2.2 中优先级问题 (复杂逻辑)

#### 2.2.1 JSON 手动拼接
**文件**: `ErpPageConfigServiceImpl.java`

**问题描述**:
- `getPageConfig()` 方法中手动拼接 JSON 字符串,逻辑复杂且容易出错
- 使用 `String.format()` 拼接多个 JSON 字段,维护困难

**代码位置**: `ErpPageConfigServiceImpl.java:243-257`

**影响范围**:
- 代码可读性差
- 容易引入 JSON 格式错误
- 难以维护

**修复建议**:
```java
// 使用 JSONObject 代替字符串拼接
public String getPageConfig(String moduleCode) {
    ErpPageConfig config = // 查询配置
    
    JSONObject result = new JSONObject();
    result.put("pageConfig", parseJson(config.getPageConfig()));
    result.put("formConfig", parseJson(config.getFormConfig()));
    result.put("tableConfig", parseJson(config.getTableConfig()));
    result.put("searchConfig", parseJson(config.getSearchConfig()));
    result.put("actionConfig", parseJson(config.getActionConfig()));
    result.put("apiConfig", parseJson(config.getApiConfig()));
    result.put("dictionaryConfig", parseJson(config.getDictConfig()));
    result.put("businessConfig", parseJson(config.getBusinessConfig()));
    result.put("detailConfig", parseJson(config.getDetailConfig()));
    result.put("moduleCode", config.getModuleCode());
    result.put("configName", config.getConfigName());
    result.put("version", config.getVersion());
    
    return result.toJSONString();
}

private Object parseJson(String jsonStr) {
    if (StringUtils.isEmpty(jsonStr)) {
        return null;
    }
    try {
        return JSON.parse(jsonStr);
    } catch (Exception e) {
        log.error("JSON 解析失败", e);
        return null;
    }
}
```

**优先级**: 🟡 中
**工作量**: 1 天

#### 2.2.2 自定义表达式引擎过于复杂
**文件**: `ComputedFieldEngine.java`

**问题描述**:
- `calculateSimpleExpression()` 方法实现了完整的四则运算解析器,逻辑复杂
- 包含括号处理、运算符优先级、递归计算等复杂逻辑
- 代码行数: 80+ 行

**代码位置**: `ComputedFieldEngine.java:308-387`

**影响范围**:
- 代码复杂度高
- 难以维护和测试
- 可能存在边界情况 bug

**修复建议**:
```java
// 方案1, 使用 Spring Expression Language (SpEL)
@Autowired
private ExpressionParser expressionParser;

public Object evaluateFormula(String formula, Map<String, Object> context) {
    Expression expression = expressionParser.parseExpression(formula);
    StandardEvaluationContext evalContext = new StandardEvaluationContext();
    
    for (Map.Entry<String, Object> entry : context.entrySet()) {
        evalContext.setVariable(entry.getKey(), entry.getValue());
    }
    
    return expression.getValue(evalContext);
}

// 方案2, 使用 Aviator 表达式引擎
<dependency>
    <groupId>com.googlecode</groupId>
    <artifactId>aviator</artifactId>
    <version>5.3.0</version>
</dependency>

public Object evaluateFormula(String formula, Map<String, Object> context) {
    for (Map.Entry<String, Object> entry : context.entrySet()) {
        AviatorEvaluator.addFunction(entry.getKey(), () -> entry.getValue());
    }
    return AviatorEvaluator.execute(formula);
}
```

**优先级**: 🟡 中
**工作量**: 2 天

### 2.3 低优先级问题 (冗余代码)

#### 2.3.1 ErpResponseUtils 冗余方法
**文件**: `ErpResponseUtils.java`

**问题描述**:
- 大量方法只是简单委托给 `ErpResponse` 的静态方法,没有增加额外价值
- 例如: `success()` 方法只是调用 `ErpResponse.ok()`,没有增加任何逻辑
- 增加了不必要的间接层,降低了代码可读性

**代码位置**: `ErpResponseUtils.java:28-227`

**影响范围**:
- 代码冗余
- 降低可读性

**修复建议**:
```java
// 删除大部分委托方法,只保留真正有价值的工具方法
// 保留:
// - errorFromException(): 从异常构建错误响应
// - logResponse(): 打印响应信息到日志
// - validateSuccessOrThrow(): 验证响应是否成功

// 删除:
// - success(): 简单委托
// - error(): 简单委托
// - warn(): 简单委托
// - fail(): 简单委托
// - ok(): 简单委托
// - conditional(): 简单委托
// - fromResult(): 简单委托
// - fromObject(): 简单委托
```

**优先级**: 🟢 低
**工作量**: 0.5 天

### 4.1 优点
- ✅ 代码结构清晰,分层合理
- ✅ 异常处理完善 (`ErpConfigExceptionHandler`)
- ✅ 日志记录详细,便于排查问题
- ✅ 使用了缓存优化性能 (`CacheUtils`)
- ✅ 配置版本管理完善

### 4.2 缺点
- ❌ 存在冗余代码 (`ErpResponseUtils`)
- ❌ 部分功能未实现 (`ErpPermissionChecker`, `ConfigRefreshListener`)

- ❌ 表达式计算过度复杂 (`ComputedFieldEngine`)
