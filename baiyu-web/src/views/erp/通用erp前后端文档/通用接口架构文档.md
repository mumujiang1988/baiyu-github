## 1. 系统架构总览

### 1.1 整体架构图

```
┌─────────────────────────────────────────────────────────┐
│                    前端 (Vue3 + Element Plus)            │
│  ┌──────────────────────────────────────────────────┐   │

│  │  - 加载配置：/erp/config/get/{moduleCode}        │   │

```

### 1.2 核心设计理念

| 理念 | 说明 | 实现方式 |
|------|------|---------|
| **配置驱动** | 页面结构、行为完全由 JSON 配置决定 | 6 个独立配置字段 |
| **通用接口** | 所有 CRUD 操作通过通用接口完成 | 构建器模式动态 SQL |
| **单一职责** | 每个配置文件只负责一个功能 | 独立文件存储 |
| **高度复用** | 一套代码支持所有业务模块 | BusinessConfigurable 模板 |
| **零硬编码** | 不写死表名、字段名、SQL | 运行时解析配置 |

## 2. 核心接口清单

### 2.1 配置管理类接口（ErpPageConfigController）

**基础路径**：`/erp/config`

| 接口 | 方法 | 权限 | 用途 | 示例 |
|------|------|------|------|------|
| `/get/{moduleCode}` | GET | `erp:config:query` | 获取完整配置 | `/erp/config/get/saleorder` |
| `/detail/{moduleCode}` | GET | `erp:config:query` | 获取详情配置 | `/erp/config/detail/saleorder` |
| `/list` | GET | `erp:config:list` | 查询配置列表 | `/erp/config/list?pageNum=1` |
| `/{configId}` | GET | `erp:config:query` | 根据 ID 获取配置 | `/erp/config/123` |
| `/` | POST | `erp:config:add` | 新增配置 | `{...}` |
| `/` | PUT | `erp:config:edit` | 修改配置 | `{...}` |
| `/{configIds}` | DELETE | `erp:config:remove` | 删除配置 | `/erp/config/123,456` |
| `/history/{configId}` | GET | `erp:config:history` | 查询历史版本 | `/erp/config/history/123` |
| `/rollback` | POST | `erp:config:rollback` | 回滚到指定版本 | `{configId, targetVersion}` |

### 2.2 引擎服务类接口（ErpEngineController）

**基础路径**：`/erp/engine`

#### 2.2.1 查询相关

| 接口 | 方法 | 权限 | 用途 | 请求体示例 |
|------|------|------|------|-----------|
| `/query/execute` | POST | `k3:{module}:query` | 执行动态查询 | ```{"moduleCode":"saleorder","tableName":"t_sale_order","queryConfig":{"defaultConditions":[{"field":"FBillNo","operator":"right_like","value":"SO2026"}]},"pageNum":1,"pageSize":10}``` |
| `/query/operators` | GET | - | 获取支持的运算符 | - |

#### 2.2.2 字典相关

| 接口 | 方法 | 权限 | 用途 | 参数示例 |
|------|------|------|------|----------|
| `/dictionary/{name}/data` | GET | `k3:{module}:query` | 加载字典数据 | `/erp/engine/dictionary/currency/data?moduleCode=saleorder` |
| `/dictionary/{name}` | GET | - | 获取缓存字典 | `/erp/engine/dictionary/orderStatus` |
| `/dictionary/cache/all` | DELETE | - | 清除所有缓存 | - |
| `/dictionary/status` | GET | - | 获取字典状态 | - |

#### 2.2.3 下推相关

| 接口 | 方法 | 权限 | 用途 |
|------|------|------|------|
| `/push/execute` | POST | `k3:{source}:push` | 执行下推 |
| `/push/preview` | POST | `k3:{source}:query` | 预览下推数据 |
| `/push/validate` | POST | `k3:{source}:query` | 验证下推数据 |
| `/push/relation/{source}/{target}` | GET | `k3:{source}:query` | 获取映射配置 |

## 3. 配置 JSON 结构详解

### 3.1 erp_page_config 表结构

```sql
CREATE TABLE `erp_page_config` (
  `config_id` BIGINT NOT NULL AUTO_INCREMENT,
  `module_code` VARCHAR(100) NOT NULL,
  `config_name` VARCHAR(200) NOT NULL,
  `config_type` VARCHAR(50) NOT NULL DEFAULT 'PAGE',
  
  -- 6 个核心 JSON 字段
  `page_config` JSON NOT NULL COMMENT '页面基础配置 (page.json)',
  `form_config` JSON COMMENT '表单 UI 组件配置 (form.json)',
  `table_config` JSON COMMENT '表格查询配置 (table.json)',
  `dict_config` JSON COMMENT '字典数据源配置 (dict.json)',
  `business_config` JSON COMMENT '业务规则配置 (config.json)',
  `detail_config` JSON COMMENT '详情页配置 (detail.json)',
  
  `version` INT NOT NULL DEFAULT 1,
  `status` CHAR(1) NOT NULL DEFAULT '1',
  PRIMARY KEY (`config_id`),
  UNIQUE KEY `uk_module_code` (`module_code`)
);
```

### 3.2 page_config（页面基础配置）

**作用**：定义页面的基本信息和主表

**JSON 结构**：
```json
{
  "pageId": "saleorder",
  "pageName": "销售订单管理",
  "permission": "k3:saleorder:query",
  "layout": "standard",
  "apiPrefix": "/erp/engine",
  "tableName": "t_sale_order"
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| pageId | String | 是 | 页面唯一标识 | "saleorder" |
| pageName | String | 是 | 页面名称 | "销售订单管理" |
| permission | String | 是 | 权限标识 | "k3:saleorder:query" |
| layout | String | 是 | 布局类型 | "standard" \| "simple" \| "complex" |
| apiPrefix | String | 是 | API 前缀 | "/erp/engine" |
| tableName | String | 是 | 主表名 | "t_sale_order" |

## 4. 通用查询接口架构

### 4.1 接口定义

**接口地址**：`POST /erp/engine/query/execute`

**权限标识**：`k3:{moduleCode}:query`

**Content-Type**：`application/json`

### 4.2 请求参数

```typescript
interface QueryRequest {
  moduleCode: string;      // 模块编码（必填）
  tableName: string;       // 表名（必填）
  queryConfig: {           // 查询配置（必填）
    enabled?: boolean;
    defaultConditions?: Condition[];
    defaultOrderBy?: OrderBy[];
  };
  pageNum: number;         // 页码，默认 1
  pageSize: number;        // 每页数量，默认 10
}

interface Condition {
  field: string;           // 字段名
  operator: string;        // 运算符
  value: any;              // 值
  description?: string;    // 描述
}

interface OrderBy {
  field: string;           // 字段名
  direction: 'ASC' | 'DESC'; // 排序方向
}
```

### 4.3 支持的运算符

| 运算符 | SQL 对应 | 示例 | 说明 |
|--------|---------|------|------|
| eq | = | `{"field":"id","operator":"eq","value":1}` | 等于 |
| ne | != | `{"field":"status","operator":"ne","value":"A"}` | 不等于 |
| gt | > | `{"field":"amount","operator":"gt","value":1000}` | 大于 |
| ge | >= | `{"field":"qty","operator":"ge","value":10}` | 大于等于 |
| lt | < | `{"field":"price","operator":"lt","value":100}` | 小于 |
| le | <= | `{"field":"discount","operator":"le","value":0.8}` | 小于等于 |
| like | LIKE '%val%' | `{"field":"name","operator":"like","value":"张"}` | 模糊查询 |
| left_like | LIKE '%val' | `{"field":"name","operator":"left_like","value":"三"}` | 左模糊 |
| right_like | LIKE 'val%' | `{"field":"billno","operator":"right_like","value":"SO"}` | 右模糊 |
| in | IN (...) | `{"field":"status","operator":"in","value":["A","B"]}` | IN 查询 |
| not_in | NOT IN (...) | `{"field":"status","operator":"not_in","value":["C"]}` | NOT IN |
| is_null | IS NULL | `{"field":"deleted","operator":"isNull"}` | 空值 |
| is_not_null | IS NOT NULL | `{"field":"email","operator":"isNotNull"}` | 非空 |
| between | BETWEEN | `{"field":"date","operator":"between","value":["2026-01-01","2026-12-31"]}` | 范围 |

### 4.4 响应格式

```typescript
interface QueryResponse {
  code: number;          // 200=成功，其他=失败
  msg: string;           // 消息
  data: {
    rows: Record<string, any>[];  // 数据列表
    total: number;                 // 总数
    moduleCode: string;           // 模块编码
  };
}
```

### 4.5 后端处理流程

```
1. 接收请求
   ↓
2. 权限检查 (ErpPermissionChecker)
   ├─ 检查 moduleCode 是否在 erp_page_config 中存在
   ├─ 检查用户是否有 k3:{moduleCode}:query 权限
   └─ 任一检查失败则返回 403
   ↓
3. 参数校验
   ├─ tableName 必须存在且非空
   ├─ queryConfig 必须存在
   └─ 参数缺失返回错误提示
   ↓
4. 构建查询条件 (buildQueryFromBuilderMode)
   ├─ 解析 defaultConditions
   ├─ 遍历每个条件
   ├─ 根据 operator 调用对应的 QueryWrapper 方法
   └─ 处理模板变量替换 (${FBillNo} → 实际值)
   ↓
5. 执行查询 (ISuperDataPermissionService)
   ├─ 应用数据权限过滤
   ├─ 应用部门权限过滤
   ├─ 执行分页查询
   └─ 返回 Page<Map<String, Object>>
   ↓
6. 数据处理 (DataProcessor)
   ├─ 计算字段处理
   ├─ 虚拟字段填充
   ├─ 字典翻译
   └─ 格式化（日期、金额等）
   ↓
7. 返回结果
```

## 5. 字典查询接口架构

### 5.1 接口定义

**接口地址**：`GET /erp/engine/dictionary/{name}/data`

**权限标识**：`k3:{moduleCode}:query`（如果传递了 moduleCode）

### 5.2 请求参数

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| name | path | 是 | 字典名称 | `/dictionary/currency/data` |
| moduleCode | query | 否 | 模块编码（用于权限检查） | `?moduleCode=saleorder` |

### 5.3 后端处理流程

```
1. 接收请求
   ↓
2. 权限检查（如果提供了 moduleCode）
   ↓
3. 从配置中读取字典定义
   ├─ 加载 dict.json
   ├─ 找到 dictionaries[name]
   └─ 未找到则返回错误
   ↓
4. 提取配置信息
   ├─ tableName: 表名
   ├─ queryConfig: 查询条件
   ├─ fieldMapping: 字段映射
   └─ cacheable: 是否缓存
   ↓
5. 构建查询条件
   └─ 使用 buildQueryFromBuilderMode()
   ↓
6. 执行查询
   ├─ 调用 ISuperDataPermissionService
   ├─ 取所有记录（LIMIT MAX_VALUE）
   └─ 应用字段映射
   ↓
7. 缓存处理（如果启用）
   ├─ 检查 Redis 缓存
   ├─ 缓存命中则直接返回
   ├─ 未命中则查询并缓存
   └─ 设置 TTL
   ↓
8. 返回结果
```

// BusinessConfigurable.vue
const loadDetailConfig = async () => {

### 7.3 常见问题与解决方案

#### 问题 1：提示"未找到配置"
**原因**：erp_page_config 表中没有对应 moduleCode 的记录

**解决**：
```sql
SELECT * FROM erp_page_config WHERE module_code = 'saleorder';
-- 如果没有，执行导入脚本
```

#### 问题 2：查询返回空数据
**原因**：
- 表名配置错误
- 查询条件过于严格
- 数据权限过滤

**排查步骤**：
```sql
-- 1. 检查配置中的表名
SELECT JSON_EXTRACT(table_config, '$.tableName') FROM erp_page_config WHERE module_code = 'saleorder';

-- 2. 直接查询表看是否有数据
SELECT * FROM t_sale_order LIMIT 10;

-- 3. 检查数据权限配置
SELECT * FROM sys_data_scope WHERE role_id IN (...);
```

#### 问题 3：字典下拉框无数据
**原因**：
- dict.json 配置错误
- 字典 API 路径错误
- 缓存问题

**解决**：
```bash
# 清除字典缓存
curl -X DELETE http://localhost:8080/erp/engine/dictionary/cache/all
```

### 8.2 前端调用完整流程

```javascript
// BusinessConfigurable.vue
import { defineComponent, ref, onMounted } from 'vue'
import axios from 'axios'

export default defineComponent({
  setup() {
    const moduleCode = ref('saleorder')
    const config = ref(null)
    const tableData = ref([])
    const loading = ref(false)
    
    // 1. 加载配置
    const loadConfig = async () => {
      try {
        const response = await axios.get(`/erp/config/get/${moduleCode.value}`)
        if (response.data.code === 200) {
          config.value = response.data.data
          
          // 解析各个配置部分
          parseConfig(response.data.data)
        }
      } catch (error) {
        console.error('加载配置失败:', error)
      }
    }
    
    // 2. 解析配置
    const parseConfig = (configData) => {
      // 解析 pageConfig
      const pageConfig = configData.pageConfig
      tableName.value = pageConfig.tableName
      
      // 解析 formConfig
      const formConfig = configData.formConfig
      formFields.value = formConfig.fields
      
      // 解析 tableConfig
      const tableConfig = configData.tableConfig
      tableColumns.value = tableConfig.columns
      queryBuilder.value = tableConfig.queryBuilder
      
      // 解析 dictConfig
      const dictConfig = configData.dictConfig
      loadDictionaries(dictConfig.dicts)
      
      // 解析 businessConfig
      const businessConfig = configData.businessConfig
      buttons.value = businessConfig.buttons
      
      // 解析 detailConfig
      const detailConfig = configData.detailConfig
      if (detailConfig) {
        loadDetailTabs(detailConfig)
      }
    }
    
    // 3. 加载字典
    const loadDictionaries = async (dicts) => {
      for (const dict of dicts) {
        const response = await axios.get(
          `/erp/engine/dictionary/${dict.dictKey}/data?moduleCode=${moduleCode.value}`
        )
        dictionaries.value[dict.dictKey] = response.data.data
      }
    }
    
    // 4. 加载主表数据
    const loadTableData = async () => {
      loading.value = true
      try {
        const queryParams = {
          moduleCode: moduleCode.value,
          tableName: tableName.value,
          queryConfig: queryBuilder.value,
          pageNum: pagination.value.pageNum,
          pageSize: pagination.value.pageSize
        }
        
        const response = await axios.post('/erp/engine/query/execute', queryParams)
        
        if (response.data.code === 200) {
          tableData.value = response.data.data.rows
          pagination.value.total = response.data.data.total
        }
      } finally {
        loading.value = false
      }
    }
    
    // 5. 加载详情标签
    const loadDetailTabs = async (detailConfig) => {
      const tabs = detailConfig.detail?.tabs || []
      
      for (const tab of tabs) {
        if (tab.type === 'table') {
          // 表格类型：调用通用查询接口
          const queryData = {
            moduleCode: moduleCode.value,
            tableName: tab.tableName,
            queryConfig: tab.queryConfig,
            pageNum: 1,
            pageSize: 1000
          }
          
          const response = await axios.post('/erp/engine/query/execute', queryData)
          tabData.value[tab.name] = response.data.data.rows
        } else if (tab.type === 'descriptions') {
          // 描述列表类型：同样调用查询接口
          const queryData = {
            moduleCode: moduleCode.value,
            tableName: tab.tableName,
            queryConfig: tab.queryConfig,
            pageNum: 1,
            pageSize: 1
          }
          
          const response = await axios.post('/erp/engine/query/execute', queryData)
          descriptionsData.value[tab.name] = response.data.data.rows[0]
        }
      }
    }
    
    // 6. 新增操作
    const handleAdd = async (formData) => {
      try {
        const response = await axios.post(
          `/erp/engine/save/${moduleCode.value}`,
          formData
        )
        
        if (response.data.code === 200) {
          ElMessage.success('新增成功')
          loadTableData() // 刷新表格
        }
      } catch (error) {
        ElMessage.error('新增失败')
      }
    }
    
    // 7. 删除操作
    const handleDelete = async (ids) => {
      try {
        const response = await axios.delete(
          `/erp/engine/delete/${moduleCode.value}`,
          { data: { ids } }
        )
        
        if (response.data.code === 200) {
          ElMessage.success('删除成功')
          loadTableData()
        }
      } catch (error) {
        ElMessage.error('删除失败')
      }
    }
    
    // 初始化
    onMounted(() => {
      loadConfig()
      loadTableData()
    })
    
    return {
      moduleCode,
      config,
      tableData,
      loading,
      loadConfig,
      loadTableData,
      handleAdd,
      handleDelete
    }
  }
})
```
