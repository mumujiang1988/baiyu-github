三、通用查询接口 完整架构逻辑（核心）
====================================================
这是系统最核心的部分，请详细解析：

1. 接口作用
   - 统一提供：页面查询、表格数据、下拉选项、弹窗数据
   - 不手写单表SQL，全部动态构造

2. 前端传参（来自JSON配置）
   - tableName  表名
   - columns    查询字段
   - conditions 查询条件（=、like、in、between、>=）
   - orderBy    排序
   - pageNum/pageSize 分页

3. 查询构造器逻辑
   - 接收JSON配置参数
   - 动态构建 MyBatis/MyBatis-Plus 动态SQL
   - 防SQL注入处理
   - 权限过滤（数据权限/部门权限）
   - 字典自动翻译

4. 接口返回
   - 列表/分页数据
   - 直接填充到表格、下拉框、选择器
   - 支持主子表关联查询

5. 支持的数据源
   - 系统字典表
   - 客户主档
   - 供应商主档
   - 商品资料
   - 国家地区
   - 业务单据表

四、JSON 配置与查询接口的联动机制
====================================================
1. JSON 中定义表名、字段、条件 → 前端读取

3. 后端查询构造器拼接SQL
4. 查询结果返回前端自动绑定到组件
5. 下拉框 → label/value 自动映射

7. 查询条件 → 自动生成查询表单
