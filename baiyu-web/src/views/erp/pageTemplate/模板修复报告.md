## 🐛 问题清单

### 1. DictionaryBuilder 导入路径错误

**错误现象**:
```

from "src\views\erp\ConfigDrivenPage\saleorder\configurable\saleorder.vue". 
Does the file exist?
```

**根本原因**:
- ❌ 错误路径：`../utils/DictionaryBuilder`（相对路径，指向不存在的目录）
- ✅ 正确位置：`@/views/erp/utils/DictionaryBuilder.js`

**影响范围**:
- `BusinessConfigurable.vue` (公共模板)
- 所有基于此模板生成的新模块

### 2. preloadApiMethods 未定义错误

**错误现象**:
```
Uncaught (in promise) ReferenceError: preloadApiMethods is not defined
    at saleorder.vue:1392:3
```

**根本原因**:
- 代码中调用了 `await preloadApiMethods()` 但该函数从未定义
- 这是一个多余的调用，实际上不需要预加载 API 方法

**影响范围**:
- `BusinessConfigurable.vue` (公共模板)
- 所有基于此模板生成的新模块

---

## 🔧 修复方案

### 修复 1: DictionaryBuilder 导入路径

**修改前**:
```javascript
import dictionaryBuilderEngine, { DictionaryBuilder } from '../utils/DictionaryBuilder'
```

**修改后**:
```javascript
import dictionaryBuilderEngine, { DictionaryBuilder } from '@/views/erp/utils/DictionaryBuilder'
```

**优势**:
- ✅ 使用项目根路径，避免相对路径计算错误
- ✅ 符合项目导入规范
- ✅ 提高代码可维护性

---

### 修复 2: 移除 preloadApiMethods 调用

**修改前**:
```javascript
onMounted(async () => {
  // 1. 先加载配置
  await initConfig()
  
  // 2. 再预加载 API 方法（配置加载完成后）
  await preloadApiMethods()  // ❌ 函数不存在
  
  // 3. 预加载字典数据
  await preloadDictionaries()
  
  // 4. 初始化引擎配置
  await initEngineConfig()
})
```

**修改后**:
```javascript
onMounted(async () => {
  // 1. 先加载配置
  await initConfig()
  
  // 2. 预加载字典数据（国家、销售人员等）
  await preloadDictionaries()
  
  // 3. 初始化引擎配置
  await initEngineConfig()
})
```

**说明**:
- ✅ 移除了不存在的函数调用
- ✅ 优化了初始化流程
- ✅ 重新编号步骤（1→2→3）

---

**文件**: `copy-module-v4.ps1`

**新增修复逻辑**（在步骤 5）:

**效果**:
