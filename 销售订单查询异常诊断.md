## 🎯 可能的原因分析

### 原因 1：后端服务未重启 

**可能性**: 90%

**分析**:
- 代码修改后，后端服务没有重新编译和重启
- Java 类加载的是旧版本的 `.class` 文件
- 旧的 `checkModulePermission()` 方法仍然只检查权限字符串

**验证方法**:
```bash
# 检查后端进程启动时间
tasklist | findstr java

# 或者在 IDEA 中查看 Console 最后启动时间
```

**解决方案**:
```bash
# 1. 停止当前运行的后端服务
# 在 IDEA 中点击停止按钮，或按 Ctrl+C

# 2. 清理并重新编译
cd d:\baiyuyunma\baiyu-github\baiyu-github\baiyu-ruoyi
mvn clean compile

# 3. 重新启动后端
# 方式一：IDEA 启动 RuoYiApplication
# 方式二：.\start-services.ps1
```

### 原因 2：moduleCode 大小写不一致 

**可能性**: 30%

**分析**:
- 数据库：`module_code = 'saleorder'` (全小写)
- 前端传递：`moduleCode = 'saleorder'` (全小写)
- 但如果前端传递的是 `'SaleOrder'` 或 `'saleOrder'`，则匹配失败

**验证方法**:
```javascript
// 在 saleorder.vue 第 604 行添加调试日志
```

```sql
FROM erp_page_config 
WHERE LOWER(module_code) = 'saleorder';
```

### 原因 4：MyBatis Plus 查询条件问题 

**可能性**: 40%

**分析**:
```java
// ErpPageConfigServiceImpl.java 第 242-246 行
ErpPageConfig config = pageConfigMapper.selectOne(new LambdaQueryWrapper<ErpPageConfig>()
    .eq(ErpPageConfig::getModuleCode, moduleCode)  // module_code = ?
```

1. 字段名映射错误（entity 的字段名与数据库不一致）

3. 数据库表被锁定

## 🎓 技术要点

### 1. MyBatis Plus LambdaQueryWrapper

```java
// 优点：类型安全，重构友好
.eq(ErpPageConfig::getModuleCode, moduleCode)

// 等价 SQL: WHERE module_code = ?

// 注意事项：
// - Entity 字段必须使用 @TableId("module_code") 注解
// - 字段名驼峰转下划线由 MyBatis Plus 自动处理
```

### 2. 权限检查三级机制

```java
// 第一级：配置存在性
if (config == null) throw CONFIG_NOT_FOUND

// 第二级：模块状态
if (!"1".equals(config.getStatus())) throw MODULE_DISABLED

// 第三级：操作权限
StpUtil.checkPermission(permission)
```

### 3. 依赖注入时机

```java
// ❌ 错误：静态方法无法使用注入的 Bean
public static void checkPermission() {...}

//  正确：通过 Spring 注入
@Component
public class ErpPermissionChecker {
    @Autowired
    private ErpPageConfigService erpPageConfigService;
    
    public void checkModulePermission(...) {...}
}
```
