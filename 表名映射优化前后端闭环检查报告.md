@Service
public class ModuleConfigService {
    
    @Autowired
    private JdbcTemplate jdbcTemplate;
    
    public String getTableNameByModule(String moduleCode) {
        String sql = "SELECT table_name FROM erp_module_config WHERE module_code = ?";
        try {
            return jdbcTemplate.queryForObject(sql, String.class, moduleCode);
        } catch (EmptyResultDataAccessException e) {
            throw new IllegalArgumentException(
                "未找到模块 [" + moduleCode + "] 对应的表名配置");
        }
    }
}

### **问题场景 3：SQL 注入风险**

**问题：** 直接使用前端传入的 `tableName` 可能导致 SQL 注入

**解决方法：** 添加白名单校验

```java
private static final Set<String> ALLOWED_TABLES = Set.of(
    "t_sale_order",
    "t_purchase_order",
    "t_inventory",
    // ... 其他允许的表名
);

@PostMapping("/query/execute")
public R<?> executeQuery(@RequestBody Map<String, Object> params) {
    String tableName = (String) params.get("tableName");
    
    // ✅ 白名单校验
    if (!ALLOWED_TABLES.contains(tableName)) {
        throw new ServiceException("非法表名：" + tableName);
    }
    
    // 安全地使用表名执行查询
}
```
