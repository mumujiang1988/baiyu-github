# 表名映射配置优化报告 - 去硬编码化

**优化时间**: 2026-03-25  
**优化目标**: 移除硬编码的模块编码与表名映射关系，改用 JSON 配置驱动

---

## 🔍 优化背景

### 原有问题

在 `SuperDataPermissionServiceImpl` 中硬编码了所有模块的表名映射：

```java
private static final Map<String, String> MODULE_TABLE_MAPPING = new HashMap<>();

static {
    MODULE_TABLE_MAPPING.put("saleOrder", "t_sale_order");           // ❌ 硬编码
    MODULE_TABLE_MAPPING.put("saleOrderEntry", "t_sale_order_entry"); // ❌ 硬编码
    MODULE_TABLE_MAPPING.put("purchaseOrder", "t_purchase_order");    // ❌ 硬编码
    // ... 更多映射
}
```

**存在的问题**：
1. ❌ **违反低代码原则** - 每新增一个模块就要修改 Java 代码
2. ❌ **维护成本高** - 需要重新编译、重启服务
3. ❌ **不灵活** - 无法动态调整表名映射
4. ❌ **容易出错** - moduleCode 大小写不一致导致查询失败（如 `saleorder` vs `saleOrder`）

---

## ✅ 优化方案

### 新架构：JSON 配置驱动

```javascript
// JSON 配置文件（数据库存储）
{
  "pageConfig": {
    "tableName": "t_sale_order"  // ✅ 直接从配置读取
  },
  "queryConfig": {
    "conditions": [...],
    "orderBy": [...]
  }
}
```

**核心思想**：
- ✅ 前端从 JSON 配置读取 `tableName`
- ✅ 直接将 `tableName` 传递给后端通用接口
- ✅ 后端不再需要 `moduleCode → tableName` 的映射

```java
    String moduleCode = (String) params.get("moduleCode");
```

**新代码（✅ 正确）**：
```java
@PostMapping("/query/execute")
public R<?> executeDynamicQuery(@RequestBody Map<String, Object> params) {
    // ✅ 直接从参数获取 tableName（前端从 JSON 配置读取）
    String tableName = (String) params.get("tableName");
    
    if (tableName == null || tableName.isEmpty()) {
        throw new ServiceException("缺少必需的参数：tableName");
    }
    
    // ✅ 直接调用基于 tableName 的方法
    Page<Map<String, Object>> page = dataPermissionService
        .selectPageByTableName(tableName, pageQuery, queryWrapper);
    
    log.info("✅ 通用查询成功，tableName: {}, total: {}", tableName, page.getTotal());
    
    return R.ok(Map.of("rows", page.getRecords(), "total", page.getTotal()));
}
```

```javascript
    data: {
```

## ⚠️ 注意事项

### 1. **旧代码迁移**
如果现有代码还在使用旧方法，会抛出异常：
```java
// ❌ 会抛出异常

// ServiceException: 表名必须从 JSON 配置获取，禁止使用模块编码映射
```

**解决方案**：改为调用新方法
```java
// ✅ 正确做法
dataPermissionService.selectPageByTableName("t_sale_order", ...);
```

### 2. **JSON 配置同步**
确保所有模块的 JSON 配置都包含 `pageConfig.tableName`：
```json
{
  "pageConfig": {
    "tableName": "t_sale_order"  // ✅ 必填
  }
}
```

### 3. **前后端协同**
前端必须先读取 JSON 配置，再将 `tableName` 传递给后端：
```javascript
// ✅ 正确流程
const config = await ERPConfigParser.loadFromDatabase(moduleCode)
const tableName = config.pageConfig.tableName

await request({
  data: { tableName, ... }
})
```
