## 🔴 问题 1：配置格式不匹配（已修复）

### 问题描述

**SQL 配置使用新格式（dictionaries 对象）：**
```json
{
  "builder": { "enabled": true },
  "dictionaries": {
    "currency": {
      "type": "dynamic",
      "config": { ... }
    },
    "nation": {
      "type": "remote",
      "config": { ... }
    }
  }
}
```

**DictionaryBuilder.buildFromConfig() 期望旧格式（dicts 数组）：**
```javascript
{
  "dicts": [
    {
      "dictKey": "currency",
      "dictType": "currency",
      "tableName": "bymaterial_dictionary"
    }
  ]
}
```

### 导致的问题

- ❌ 无法从 SQL 配置加载字典
- ❌ 前端报错：`dictionaryConfig.dicts is undefined`
- ❌ 所有动态字典无法注册

### 修复方案

**修改文件**: `src/views/erp/utils/DictionaryBuilder.js`

**修改内容**:
```javascript
// ✅ 修改前：期望 dicts 数组
const dicts = dictionaryConfig.dicts || []
for (const dict of dicts) {
  const { dictKey, dictType, data, tableName, searchable } = dict
  // ...
}

// ✅ 修改后：支持 dictionaries 对象
const dictionaries = dictionaryConfig.dictionaries || {}
for (const [dictName, dictConfig] of Object.entries(dictionaries)) {
  const { type, data, table, config } = dictConfig
  // ...
}
```

**关键变更点**:
1. 从遍历数组改为遍历对象 entries
2. 使用 `type` 字段判断字典类型（static/dynamic/remote）
3. 优先使用配置中的 `config.api`，降级到统一接口
4. 特殊处理国家字典的搜索 API

## 🔴 问题 2：preloadAll 方法不支持新格式（已修复）

### 问题描述

**原代码问题**:
- 硬编码 API 路径 `/erp/engine/dict/union/{name}`
- 忽略配置中的自定义 API
- 错误处理不完善
- 日志信息不清晰

### 修复方案

**修改内容**:
```javascript
// ✅ 增强版 preloadAll
async preloadAll(moduleCode) {
  const promises = []
  
  this.registry.forEach((builder, name) => {
    // ✅ 只预加载 dynamic 类型，remote 类型按需加载
    if (builder.type === 'dynamic') {
      const promise = builder.load(async () => {
        // ✅ 优先使用配置中的 API
        const apiUrl = builder.config?.api || `/erp/engine/dict/union/${name}`
        
        console.log(`🌐 加载字典：${name}, API: ${apiUrl}`)
        
        const response = await request({ url: apiUrl, method: 'get' })
        
        let data = []
        if (response.code === 200 || response.errorCode === 0) {
          // ✅ 处理分段返回的数据
          const result = response.data || response
          data = [
            ...(result.dictTypeList || []),
            ...(result.dictDataList || [])
          ]
          console.log(
            `✅ 字典加载成功：${name}, ` +
            `dictTypeList: ${result.dictTypeList?.length || 0}, ` +
            `dictDataList: ${result.dictDataList?.length || 0}, ` +
            `总计：${data.length}条`
          )
        } else if (Array.isArray(response)) {
          // ✅ 兼容旧接口
          data = response
          console.log(`✅ 字典加载成功（数组格式）：${name}, 共 ${data.length} 条`)
        } else {
          console.warn(`⚠️ 字典加载返回格式异常：${name}`, response)
        }
        
        return data
      })
      promises.push(promise)
    } else if (builder.type === 'remote') {
      // ✅ 远程搜索字典不预加载，按需加载
      console.log(`⏭️ 跳过远程搜索字典预加载：${name}（按需加载）`)
    }
  })
  
  if (promises.length > 0) {
    await Promise.all(promises)
    console.log('🎉 字典预加载完成')
  } else {
    console.log('ℹ️ 没有需要预加载的字典')
  }
}
```

**改进点**:
1. ✅ 支持配置中的自定义 API
2. ✅ 增强日志输出，显示详细加载信息
3. ✅ 添加分段数据统计
4. ✅ 区分 dynamic 和 remote 类型的加载策略
5. ✅ 完善的错误处理和格式兼容性

## 🔴 问题 3：国家搜索逻辑不兼容新配置（已修复）

### 问题描述

**原代码问题**:
```javascript
// ❌ 访问旧的配置结构
const nationDict = BusinessTemplate.value.dictionaryConfig?.nation
if (nationDict && nationDict.api) {
  // 旧的路径匹配逻辑
  const searchUrl = nationDict.api.includes('/search') 
    ? `${nationDict.api}?keyword=${encodeURIComponent(keyword)}`
    : `${nationDict.api.replace('{moduleCode}', getModuleCode())}&keyword=${encodeURIComponent(keyword)}`
}
```

**问题点**:
- 配置路径错误：应该是 `dictionaryConfig.dictionaries.nation`
- 未使用 `type: 'remote'` 判断
- URL 拼接逻辑复杂且易错

### 修复方案

**修改文件**: `src/views/erp/pageTemplate/configurable/BusinessConfigurable.vue`

**修改内容**:
```javascript
// ✅ 新的国家搜索逻辑
const searchNations = async (keyword) => {
  if (!keyword || keyword.trim() === '') {
    nationOptions.value = []
    return
  }
  
  nationSearchLoading.value = true
  try {
    // ✅ 从配置中获取国家字典的配置
    const dictConfig = BusinessTemplate.value.dictionaryConfig?.dictionaries?.nation
    
    if (dictConfig && dictConfig.type === 'remote') {
      // ✅ 使用配置中的搜索 API
      const searchApi = dictConfig.config?.searchApi || 
        '/erp/engine/country/search?keyword={keyword}&limit=20'
      const searchUrl = searchApi.replace('{keyword}', encodeURIComponent(keyword))
      
      console.log(`🔍 搜索国家：${keyword}, URL: ${searchUrl}`)
      
      const response = await request(searchUrl)
      
      let data = []
      if (response.code === 200 || response.errorCode === 0) {
        data = response.data || response.rows || []
      } else if (Array.isArray(response)) {
        data = response
      }
      
      // ✅ 映射为标准格式 { label, value }
      nationOptions.value = data.map(item => ({
        label: item.labelZh || item.name || item.label,
        value: item.id || item.kingdee || item.value,
        labelEn: item.labelEn || item.name_en // 保留英文用于扩展
      }))
      
      console.log(`✅ 国家搜索结果：${nationOptions.value.length} 条`)
    } else {
      console.warn('⚠️ 国家字典未配置或类型不正确')
      nationOptions.value = []
    }
  } catch (error) {
    console.warn('搜索国家失败:', error.message)
    nationOptions.value = []
  } finally {
    nationSearchLoading.value = false
  }
}
```

**改进点**:
1. ✅ 正确的配置访问路径
2. ✅ 使用 `type === 'remote'` 判断
3. ✅ 简化的 URL 拼接逻辑
4. ✅ 更清晰的字段映射
5. ✅ 增强的错误处理和日志

### 2. 添加字典加载重试机制
```javascript
async loadWithRetry(loader, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      return await loader()
    } catch (error) {
      if (i === retries - 1) throw error
      console.warn(`加载失败，第 ${i + 1} 次重试...`)
      await new Promise(r => setTimeout(r, 1000 * (i + 1)))
    }
  }
}
```

### 3. 添加字典健康检查
```javascript
export function checkDictionaryHealth(dictName) {
  const builder = dictionaryBuilderEngine.get(dictName)
  if (!builder) {
    return { status: 'not_registered', healthy: false }
  }
  
  const status = builder.getStatus()
  return {
    status: status.exists ? 'registered' : 'not_exists',
    healthy: status.loaded && !status.expired && !status.error,
    ...status
  }
}
```
