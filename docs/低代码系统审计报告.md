##### 🔴 问题7: 运算符优先级处理错误

**严重等级**: 🔴 **高危**  
**代码位置**: `ComputedFieldEngine.java:311-342`

**问题代码**:
```java
private BigDecimal calculateSimpleExpression(String expression) {
    // 处理加减法(最低优先级)
    int plusIndex = findOperatorIndex(expression, '+');
    if (plusIndex > 0) {
        BigDecimal left = calculateSimpleExpression(expression.substring(0, plusIndex));
        BigDecimal right = calculateSimpleExpression(expression.substring(plusIndex + 1));
        return left.add(right);
    }
    
    int minusIndex = findOperatorIndex(expression, '-');
    if (minusIndex > 0) {
        BigDecimal left = calculateSimpleExpression(expression.substring(0, minusIndex));
        BigDecimal right = calculateSimpleExpression(expression.substring(minusIndex + 1));
        return left.subtract(right);
    }
    
    // 处理乘除法(高优先级)
    int multiplyIndex = expression.indexOf('*');
    if (multiplyIndex > 0) {
        BigDecimal left = calculateSimpleExpression(expression.substring(0, multiplyIndex));
        BigDecimal right = calculateSimpleExpression(expression.substring(multiplyIndex + 1));
        return left.multiply(right);
    }
    // ...
}
```

**问题分析**:
- 代码注释说"加减法最低优先级",但实际先处理加减
- 导致运算优先级错误: `2 + 3 * 4` 应该等于 `14`,但可能计算为 `20`
- **这是严重的业务逻辑错误,会导致财务计算错误**

**影响场景**:
- 订单金额计算: `单价 * 数量 + 运费`
- 成本核算: `材料成本 + 人工成本 * 系数`
- 利润计算: `销售额 - 成本 * 税率`

**修复优先级**: 🔴 **P0 - 立即修复**

##### 🔴 问题13: get方法重复定义导致栈溢出

**严重等级**: 🔴 **高危**  
**代码位置**: `DictionaryBuilder.js:224, 266`

**问题代码**:
```javascript
class DictionaryBuilderEngine {
  // ...

  /**
   * 获取字典构建器
   */
  get(name) { // 第一次定义
    const builder = this.registry.get(name)
    if (!builder) {
      console.warn(` 字典未注册：${name}`)
    }
    return builder
  }

  // ...

  /**
   * 获取字典数据
   */
  get(name) { // ❌❌ 第二次定义,覆盖了第一个
    const builder = this.get(name) // ❌❌❌ 递归调用自己,导致栈溢出
    if (!builder) {
      return []
    }
    return builder.get()
  }
}
```

**问题分析**:
1. 第224行定义了第一个`get(name)`方法,用于获取字典构建器
2. 第266行又定义了第二个`get(name)`方法,覆盖了第一个
3. 第二个方法内部调用`this.get(name)`,递归调用自己
4. **导致无限递归,栈溢出崩溃**

**影响范围**:
- **所有使用字典的前端页面都会崩溃**
- 销售订单、采购订单、库存管理等核心页面
- **这是致命Bug,必须立即修复**

**修复优先级**: 🔴 **P0 - 立即修复**

**修复方案**: 重命名第二个get方法
```javascript
// 修改第266行
getData(name) { // 改名为getData
    const builder = this.get(name) // 调用第一个get方法
    if (!builder) {
      return []
    }
    return builder.get()
}
```

### 7.2 边界条件未处理

#### 1. 空值处理不一致

**问题**: 部分方法返回null,部分返回空集合

**示例**:

**影响**: 调用方需要同时处理null和空集合,增加复杂度

#### 2. 类型转换异常

**问题**: 字符串转数字未捕获NumberFormatException

**示例**:
```java

```

**影响**: 非数字字符串会导致系统崩溃

#### 3. 并发安全问题

**问题**: 部分缓存操作未加锁

**示例**:
```java
// DictionaryBuilderEngine.java
public void clear(String name) {
    DictionaryCache cache = dictionaryCache.get(name);
    if (cache != null) {
        cache.loaded = false; // ❌ 无锁操作

        cache.timestamp = 0;
    }
}
```

**影响**: 并发修改可能导致数据不一致

### 7.3 异常处理建议

1. **统一返回值**: 空集合统一返回`Collections.emptyList()`
2. **异常捕获**: 所有可能抛出异常的操作都要捕获
3. **并发安全**: 使用同步块或ConcurrentHashMap的原子操作
4. **防御性编程**: 对所有外部输入进行校验

### 12.2 核心优势

####  架构设计

1. **低代码理念先进**: 配置驱动,灵活度高
2. **前后端分离清晰**: 职责明确,易于维护
3. **引擎化设计**: 查询引擎、下推引擎、计算引擎等模块化设计

####  功能实现

1. **字典构建器优雅**: 支持静态、动态、远程搜索三种类型
2. **虚拟字段优化**: 批量查询避免N+1问题
3. **配置解析完善**: 支持多种数据格式,容错性好

####  性能优化

1. **缓存机制**: 前后端都实现了缓存
2. **并行查询**: 使用Promise.all并行查询子表格
3. **批量操作**: 虚拟字段批量查询优化到位

---

### 12.3 核心风险

#### ❌ 安全风险

1. **SQL注入风险高**: 2处SQL注入漏洞,可能导致数据泄露
2. **前端崩溃风险**: DictionaryBuilder.js栈溢出Bug会导致所有页面崩溃
3. **XSS风险**: 模板变量替换未转义

#### ❌ 业务风险

1. **计算错误**: ComputedFieldEngine运算优先级错误,会导致财务数据错误
2. **下推失败**: PushDownEngine除零异常会导致业务流程中断

#### ❌ 质量风险

1. **测试覆盖率0%**: 无单元测试,质量保障不足
2. **代码冗余25%**: 维护成本高,Bug风险大
3. **边界条件未处理**: 空值、异常、并发问题多
