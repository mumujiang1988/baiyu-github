# 销售订单配置优化方案 - 以数据库字段名为准

## 📋 **问题概述**

### **P0: 字段命名不一致导致查询条件传递错误**

**当前问题**:
- ❌ 前端配置使用小写驼峰 (`fseller`, `fbillno`)
- ❌ 数据库字段使用金蝶风格 (`FSalerId`, `FBillNo`)
- ❌ 查询条件传递时字段名不匹配，导致查询失败

---

## 🎯 **优化原则**

### **核心策略：以数据库字段名为准**

| 层次 | 命名规范 | 示例 | 说明 |
|------|---------|------|------|
| **数据库层** | 金蝶 K3 风格 | `FSalerId`, `FBillNo` | ✅ **基准**，不可更改 |
| **后端层** | 金蝶 K3 风格 | `FSalerId`, `FBillNo` | ✅ 与数据库一致 |
| **前端配置层** | 金蝶 K3 风格 | `FSalerId`, `FBillNo` | ✅ **统一为数据库风格** |
| **前端绑定层** | 小写驼峰 | `formData.fseller` | ⚠️ 仅表单绑定使用小写 |

## 🔧 **字段映射规则总结**

### **规则 1: 数据库字段优先**

```javascript
// ✅ 推荐：配置中直接使用数据库字段名
const config = {
  field: "FSalerId",      // 数据库字段名
  dbField: "FSalerId",    // 显式指定
  formBind: "fseller"     // 表单绑定使用小写（仅当需要时）
}
```

### **规则 2: 虚拟字段特殊处理**

```javascript
// ⚠️ 虚拟字段（数据库不存在）需要额外配置
const virtualField = {
  field: "orderStatus",   // 前端显示的字段名
  virtual: true,          // 标记为虚拟
  sourceField: "FStatus", // 数据来源
  transform: {            // 值转换逻辑
    type: "map",
    mapping: {
      "1": "待提交",
      "2": "已提交",
      "3": "已审核"
    }
  }
}
```

### **规则 3: 大小写敏感**

```javascript
// ⚠️ 注意：数据库字段名严格区分大小写
{
  "field": "F_ora_BaseProperty",  // ✅ 正确（与数据库一致）
  "field": "f_ora_baseproperty"   // ❌ 错误（大小写不匹配）
}
```

## ⚠️ **注意事项**

### **1. 向后兼容性**

如果现有代码已经使用小写字段名，需要：
- ✅ 在查询构建器中添加字段映射逻辑
- ✅ 保持表单绑定使用小写（`formData.fseller`）
- ✅ 仅在配置层面改为数据库字段名

### **2. 虚拟字段处理**

对于 `orderStatus` 等虚拟字段：
- ✅ 添加 `virtual: true` 标记
- ✅ 指定 `sourceField` 数据来源
- ✅ 配置 `transform` 值转换逻辑

### **3. 大小写敏感**

数据库字段名严格区分大小写：
- ✅ `F_ora_BaseProperty` (正确)
- ❌ `f_ora_baseproperty` (错误)
- ❌ `F_ORA_BASEPROPERTY` (错误)
