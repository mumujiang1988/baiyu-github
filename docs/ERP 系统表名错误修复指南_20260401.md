## 🔍 **根本原因分析**

### **1. 付款申请单 - 表完全不存在**

**问题**: `f_rectunit_type` 表在数据库中不存在

**可能原因**:
- 建表脚本未执行
- 表名拼写错误
- 使用了错误的表名（应该是 `payment_apply` 或其他）

**解决方案**:
```sql
-- 步骤 1: 检查数据库中是否存在该表
SELECT TABLE_NAME, TABLE_COMMENT 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'test' 
AND TABLE_NAME LIKE '%payment%' OR TABLE_NAME LIKE '%rectunit%';

-- 步骤 2: 如果表不存在，需要执行建表脚本
-- 查找并执行：付款申请单建表.sql 或 payment_apply.sql
```

---

### **2. 检验单 - 表名前缀不一致**

**问题**: 配置中使用 `receipt_notice_full`，但后端查询 `t_receipt_notice_full`

**可能原因**:
- 前端配置和后端实现使用的表名不一致
- 缺少 `t_` 前缀

**解决方案**:
```sql
-- 步骤 1: 检查哪个表存在
SELECT TABLE_NAME, TABLE_COMMENT 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'test' 
AND TABLE_NAME IN ('receipt_notice_full', 't_receipt_notice_full');

-- 步骤 2A: 如果 t_receipt_notice_full 存在，修改前端配置
UPDATE erp_page_config
SET 
  page_config = JSON_REPLACE(page_config, '$.tableName', 't_receipt_notice_full'),
  form_config = JSON_REPLACE(form_config, '$.formConfig.tableName', 't_receipt_notice_full'),
  table_config = JSON_REPLACE(table_config, '$.tableName', 't_receipt_notice_full')
WHERE module_code = 'inspection';

-- 步骤 2B: 如果 receipt_notice_full 存在，修改后端代码
-- 将 t_receipt_notice_full 改为 receipt_notice_full
```

---

### **3. 采购入库单 - 大小写敏感问题**

**问题**: 配置中是 `purchase_instock`（小写），但错误信息显示 `PurchaseInStock`（驼峰）

**可能原因**:
- MySQL 在 Linux 下表名大小写敏感
- 某个地方使用了驼峰命名

**解决方案**:
```sql
-- 步骤 1: 确认表的实际名称
SELECT TABLE_NAME, TABLE_COMMENT 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'test' 
AND TABLE_NAME LIKE '%purchase%instock%';

-- 步骤 2: 如果是大小写问题，重命名表
RENAME TABLE PurchaseInStock TO purchase_instock;

-- 或者修改配置（如果表是小写的）
UPDATE erp_page_config
SET 
  page_config = JSON_REPLACE(page_config, '$.tableName', 'purchase_instock')
WHERE module_code = 'purchaseinstock';
```

## 📝 **预防措施**

### **1. 统一命名规范**

```javascript
// ✅ 正确：全部使用 snake_case（小写 + 下划线）
{
  "tableName": "purchase_instock",
  "tableName": "t_sale_order",
  "tableName": "f_payment_apply"
}

// ❌ 错误：不要使用驼峰或混合大小写
{
  "tableName": "PurchaseInStock",      // 驼峰
  "tableName": "t_SaleOrder",          // 混合
  "tableName": "F_PaymentApply"        // 大写 + 下划线
}
```

### **2. 添加表名前缀规则**

| 类型 | 前缀 | 示例 |
|------|------|------|
| **主表** | `t_` | `t_sale_order`, `t_purchase_order` |
| **财务表** | `f_` | `f_payment_apply`, `f_receive_bill` |
| **关联表** | 无/描述性 | `purchase_instock_entry`, `sales_order_detail` |

### **3. 建表脚本标准化**

```sql
-- 标准建表脚本模板
CREATE TABLE IF NOT EXISTS `t_table_name` (
  `id` BIGINT NOT NULL COMMENT '主键 ID',
  `FBillNo` VARCHAR(100) COMMENT '单据编号',
  -- ... 其他字段
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_general_ci COMMENT='表说明';
```
