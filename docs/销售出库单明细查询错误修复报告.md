## 🐛 **根本原因**

### **字段名大小写不一致**

```json
// ❌ 错误配置（使用 PascalCase）
"defaultOrderBy": [{"field": "FMaterialID", "direction": "ASC"}]

// ✅ 正确配置（应使用 snake_case）
"defaultOrderBy": [{"field": "f_material_id", "direction": "ASC"}]
```

### **技术背景**

MySQL 字段命名规范：
- ✅ **snake_case**: `f_material_id`, `f_bill_no`, `f_create_time`
- ❌ **PascalCase**: `FMaterialID`, `FBillNo`, `FCreateTime`

当配置中使用错误的字段名时：
1. MyBatis-Plus 会直接拼接 SQL
2. MySQL 找不到 `FMaterialID` 字段
3. 抛出 `BadSqlGrammarException`

## 🎯 **最佳实践建议**

### **1. 字段命名规范**

```javascript
// ✅ 始终使用 snake_case（数据库字段风格）
{
  field: "f_material_id",      // 正确
  operator: "eq"
}

// ❌ 禁止使用 PascalCase（Java 实体风格）
{
  field: "FMaterialID",        // 错误
  operator: "eq"
}
```

### **2. 配置检查技巧**

**方法 1：搜索关键字**
```bash
# 在所有 SQL 文件中搜索 PascalCase 字段
grep -r "F[A-Z][a-z]*[A-Z]" *.sql
```

**方法 2：对比表结构**
```sql
-- 查看实际表结构
DESCRIBE f_sal_outbound_details;

-- 对比配置中的字段名是否存在
SELECT COLUMN_NAME 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = 'f_sal_outbound_details';
```

### **3. 预防措施**

**建议 1：创建配置校验工具**
```javascript
// config-validator.js
function validateFieldNames(config) {
  const fields = config.detail.tabs[0].table.columns.map(col => col.prop);
  const orderByFields = config.detail.tabs[0].queryConfig.defaultOrderBy.map(o => o.field);
  
  // 检查是否所有字段都使用 snake_case
  const invalidFields = [...fields, ...orderByFields].filter(f => /[A-Z]/.test(f));
  
  if (invalidFields.length > 0) {
    console.error('发现 PascalCase 字段:', invalidFields);
    return false;
  }
  return true;
}
```

**建议 2：添加 CI/CD 检查**
```yaml
# .github/workflows/config-check.yml
name: ERP Config Validator
on: [push]
jobs:
  check-field-names:
    runs-on: ubuntu-latest

      - name: Check PascalCase in configs

          grep -r "F[A-Z][a-z]*[A-Z]" src/views/erp/erpDOC/sql/*.sql && \
          echo "❌ Found PascalCase field names!" && \

          echo "✅ All field names are correct"
```
