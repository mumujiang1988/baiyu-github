#### ❌ **异常情况** (可能的问题):

##### 问题 1: 查询按钮点击后无日志
```
(没有任何日志输出)
```
**原因**: 按钮事件未绑定或组件未正确加载  
**解决**: 检查页面是否正确初始化，组件是否挂载成功

---

##### 问题 2: 字段值为空
```
🔍 [查询构建] 处理字段：fbill_no 组件类型：input
  - 字段 fbill_no:
    原始值：undefined  // ❌ 这里是 undefined 或 null
    ⚠️ 值为空，跳过
```
**原因**: 
- Input 组件的 v-model 未正确绑定
- 字段名不匹配 (`fbill_no` vs `FBillNo`)

**解决**: 
1. 检查 `queryParams.value` 中是否有 `fbill_no` 字段
2. 对比配置中的 `field` 字段名

---

##### 问题 3: 查询条件为空数组
```
🎯 [查询构建] 最终查询条件：[]  // ❌ 空数组
```
**原因**: 所有字段都被跳过了  
**解决**: 检查每个字段的值是否正确传递

---

##### 问题 4: 请求未发出
```
🚀 [查询按钮] 准备执行查询...
⏱️ [查询按钮] 延迟执行 getList()
(然后没有然后了...)
```
**原因**: 
- `loading.value` 为 true，被拦截
- setTimeout 未执行

**解决**: 检查 `loading` 状态是否正常重置

---

##### 问题 5: 后端返回错误
```
🌐 [数据查询] 发送请求到:/erp/engine/query/execute
❌ [数据查询] 查询失败：Error: ...
❌ [数据查询] 错误详情：SQL syntax error...
```
**原因**: 后端 SQL 执行失败  
**解决**: 
1. 查看错误详情
2. 检查 SQL 语法
3. 查看后端日志

## 📊 常见场景分析

### 场景 A: 供应商查询正常，单据编号查询无反应

**预期日志**:
```
✅ 供应商查询:
  - 字段 fsupplier_id:
    原始值："SUP001"
    ✅ 添加查询条件

❌ 单据编号查询:
  - 字段 fbill_no:
    原始值：undefined  // ❌ 问题在这里!
    ⚠️ 值为空，跳过
```

**诊断**: `queryParams.value.fbill_no` 未正确赋值  
**解决**: 检查 Input 组件的 v-model 绑定

---

### 场景 B: 查询按钮完全无反应

**预期日志**:
```
(无任何日志)
```

**诊断**: 按钮 click 事件未触发  
**解决**:
1. 检查按钮是否被禁用
2. 检查事件绑定是否正确
3. 检查组件是否挂载成功

---

### 场景 C: 查询后表格无数据

**预期日志**:
```
🌐 [数据查询] 发送请求到:/erp/engine/query/execute
✅ [数据查询] 收到响应：{ rows: [], total: 0 }
```

**诊断**: 后端返回空数据  
**解决**:
1. 检查数据库中是否有匹配的数据
2. 检查查询条件是否正确
3. 直接执行 SQL 验证:
```sql
SELECT * FROM po_order_bill_head WHERE fbill_no LIKE '%PO%';
```
