**异常现象**: 销售订单详情页的"成本暂估"页签显示数据为空

**影响范围**: 所有销售订单的成本明细查询

---

## 🔍 **问题诊断**

### **核心问题 (3 处)**

#### 1. SQL 配置中的查询条件字段错误 ❌
**文件**: `baiyu-web/src/views/erp/脚本库/销售订单初始化配置.sql`

**位置**: 第 593 行

**问题**:
```json
{
  "field": "fbillno",  // ❌ 小写
  "operator": "eq",
  "value": "${billNo}"
}
```

**数据库实际字段**: `FBillNo` (大写 F)

---

#### 2. 排序字段不存在 ❌
**文件**: 同上

**位置**: 第 599 行

**问题**:
```json
"defaultOrderBy": [{"field": "fentryid", "direction": "ASC"}]
```

**实际情况**: 
- `t_sale_order_cost` 表中**没有 `fentryid` 字段**
- 正确的主键是：`FID`

---

#### 3. multiTableQueryBuilder.js 中的硬编码错误 ❌
**文件**: `baiyu-web/src/views/erp/pageTemplate/utils/multiTableQueryBuilder.js`

**位置**: 第 270 行、第 277 行

**问题**:
```javascript
// 成本表查询
conditions: [{
  field: 'order_id',  // ❌ 错误的字段名
  operator: 'eq',
  value: billNo
}],
orderBy: [{
  field: 'id',  // ❌ 不存在的字段
  direction: 'ASC'
}]
```

**应该使用**: 
- 查询字段：`FBillNo`
- 排序字段：`FID`

## ✅ **修复方案**

### **修复文件 1: 销售订单初始化配置.sql**

**修改位置**: 第 593 行、第 599 行

**修改前**:
```json

  "defaultConditions": [{
    "field": "fbillno",
    "operator": "eq",
    "value": "${billNo}"
  }],
  "defaultOrderBy": [{"field": "fentryid", "direction": "ASC"}]
}
```

**修改后**:

## ⚠️ **注意事项**

1. **字段命名规范**: 
   - 数据库字段名通常为大写开头 (如 `FBillNo`, `FID`)
   - Java 实体属性名为小写开头 (如 `fbillno`, `fid`)
   - 查询条件中应使用**数据库字段名**

2. **动态查询引擎**: 
   - 后端 `SqlBuilder` 会将字段名直接拼接到 SQL 中
   - 字段名必须与数据库表结构一致
   - 已有防 SQL 注入校验 (正则白名单)

3. **模板变量替换**:
   - `${billNo}` 会在运行时被替换为实际单据编号
   - 确保模板变量名称与上下文数据 key 一致
