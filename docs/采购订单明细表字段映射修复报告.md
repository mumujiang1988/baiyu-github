**状态**: ✅ 已修复

## 🔍 问题现象

```
Failed to load sub-table data: 查询失败:SQL 语法错误，请联系管理员检查配置:
PreparedStatementCallback; bad SQL grammar [SELECT * FROM po_order_bill_head_entry 
WHERE FInterID = ? ORDER BY FEntryID DESC LIMIT ? OFFSET ?]
```

## 📋 根本原因

**字段命名风格不一致** - 配置使用了金蝶 K/3 Cloud 风格的大写字段名，但实际数据库表结构使用小写字段名。

### 字段映射对比表

| ❌ 配置中的错误字段 | ✅ 实际数据库字段 | 📝 字段说明 |
|-------------------|-----------------|-----------|
| `FInterID` | `fid` | 主表 ID (关联字段) |
| `FEntryID` | `id` | 明细主键 |
| `FMaterialId` | `f_gyswlbm` | 供应商物料编码 |
| `FMaterialName` | `f_gyswlmc` | 供应商物料名称 |
| `FQty` | `fqty` | 采购数量 |
| `FPrice` | `fprice` | 单价 |
| `FAmount` | `fentryamount` | 金额 |
| `FUnitId` | `funitid` | 单位 |
| `FStockId` | ❌ **缺失** | 表中没有仓库字段 |
| `FRemark` | `fentrynote` | 备注 |

## ✅ 修复内容

### 1. queryConfig 修复

**修改前:**
```json
"queryConfig": {
  "defaultConditions": [{
    "field": "FInterID",
    "operator": "eq",
    "value": "${interId}",
    "description": "按订单 ID 查询明细"
  }],
  "defaultOrderBy": [{"field": "FEntryID", "direction": "ASC"}]
}
```

**修改后:**
```json
"queryConfig": {
  "defaultConditions": [{
    "field": "fid",
    "operator": "eq",
    "value": "${fbill_no}",
    "description": "按采购订单 ID 查询明细"
  }],
  "defaultOrderBy": [{"field": "id", "direction": "ASC"}]
}
```

**关键变化:**
- `FInterID` → `fid` (使用实际关联字段)
- `${interId}` → `${fbill_no}` (使用正确的模板变量)
- `FEntryID` → `id` (排序字段修正)

## 📚 经验教训

### ERP 配置字段命名规范

1. **必须使用数据库实际字段名** - 不能照搬金蝶 K/3 Cloud 的字段风格
2. **大小写敏感** - MySQL 在 Windows 上不敏感，但为了跨平台兼容性，统一使用小写
3. **先查表结构再配置** - 配置前必须先确认数据库表的实际字段
4. **模板变量要匹配** - `${fbill_no}` 对应主表字段名

### 配置检查清单

在配置详情页签时，需要检查:
- [ ] `relationConfig.masterField` - 主表关联字段
- [ ] `relationConfig.detailField` - 明细表关联字段
- [ ] `queryConfig.defaultConditions[].field` - 查询条件字段
- [ ] `queryConfig.defaultConditions[].value` - 模板变量
- [ ] `table.columns[].prop` - 表格列字段
- [ ] 所有字段都在数据库表结构中存在

## 🔄 第二次修复：relationConfig 关联字段修正

### ❌ 发现的问题

在审计中发现 `detail_config` 的 `relationConfig` 配置存在严重问题:

```json
// 错误配置
"relationConfig": {
  "masterField": "fbill_no",     // ❌ 使用业务主键
  "detailField": "fbillNo"       // ❌ 字段名大小写不一致
}
```

**问题分析:**
1. **主表主键是 `id`**,不是 `fbill_no`
2. **明细表关联字段是 `fid`**,不是 `fbillNo`
3. **同时启用了 queryConfig**,导致双重过滤冲突

### ✅ 修复内容

**修改前:**
```json
"relationConfig": {
  "masterTable": "po_order_bill_head",
  "masterField": "fbill_no",
  "detailTable": "po_order_bill_head_entry",
  "detailField": "fbillNo"
},
"queryConfig": {
  "enabled": true,
  "defaultConditions": [{
    "field": "fid",
    "value": "${fbill_no}"
  }]
}
```

**修改后:**
```json
"relationConfig": {
  "masterTable": "po_order_bill_head",
  "masterField": "id",           // ✅ 改为使用主表主键
  "detailTable": "po_order_bill_head_entry",
  "detailField": "fid",          // ✅ 使用明细表实际关联字段
  "operator": "eq"
},
"queryConfig": {
  "enabled": false               // ✅ 禁用，避免双重过滤
}
```

### 🎯 修复效果

修复后的 SQL 查询应该是:
```sql
-- relationConfig 生成的 SQL
SELECT * FROM po_order_bill_head_entry 
WHERE fid = ? 
ORDER BY id ASC 
LIMIT ? OFFSET ?
```

**预期结果:**
- ✅ 详情页签可以正常加载明细数据
- ✅ 关联关系正确 (主表.id → 明细表.fid)
- ✅ 无双重过滤冲突
