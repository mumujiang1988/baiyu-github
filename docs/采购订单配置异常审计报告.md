### 2️⃣ detail_config 配置深度审计

#### ❌ **严重问题 1**: relationConfig 与 queryConfig 冲突

**当前配置:**
```json
{
  "relationConfig": {
    "enabled": true,
    "masterTable": "po_order_bill_head",
    "masterField": "fbill_no",
    "detailTable": "po_order_bill_head_entry",
    "detailField": "fbillNo",
    "operator": "eq"
  },
  "queryConfig": {
    "enabled": true,
    "defaultConditions": [
      {
        "field": "fid",
        "operator": "eq",
        "value": "${fbill_no}"
      }
    ]
  }
}
```

##### 🔴 **问题分析**

1. **双重过滤机制冲突**:
   - `relationConfig` 会生成：`WHERE fbillNo = ?`
   - `queryConfig` 会生成：`AND fid = ?`
   - **两个条件同时生效，可能导致查询失败**

2. **字段引用混乱**:
   - `relationConfig.detailField = "fbillNo"` (使用明细表字段)
   - `queryConfig.field = "fid"` (使用主表 ID 字段)
   - **模板变量 `${fbill_no}` 是主表字段，不能用于明细表查询**

3. **正确的关联逻辑**:
   ```sql
   -- relationConfig 生成的 SQL
   SELECT * FROM po_order_bill_head_entry 
   WHERE fbillNo = ?  -- ❌ 错误：应该用 fid
   
   -- queryConfig 生成的 SQL  
   AND fid = ?        -- ❌ 错误：重复过滤
   ```

##### ✅ **修复方案**

**方案 A: 仅使用 relationConfig (推荐)**
```json
{
  "relationConfig": {
    "enabled": true,
    "masterTable": "po_order_bill_head",
    "masterField": "id",          // ✅ 改为使用主表主键
    "detailTable": "po_order_bill_head_entry",
    "detailField": "fid",         // ✅ 使用明细表关联字段
    "operator": "eq"
  },
  "queryConfig": {
    "enabled": false              // ✅ 禁用 queryConfig
  }
}
```

**方案 B: 仅使用 queryConfig**
```json
{
  "relationConfig": {
    "enabled": false              // ✅ 禁用 relationConfig
  },
  "queryConfig": {
    "enabled": true,
    "defaultConditions": [
      {
        "field": "fid",
        "operator": "eq",
        "value": "${id}"          // ✅ 使用主表主键值
      }
    ]
  }
}
```

#### ❌ **严重问题 2**: 模板变量引用错误

**问题代码:**
```json
"value": "${fbill_no}"
```

**分析:**
- `${fbill_no}` 引用的是**主表当前行的 fbill_no 字段值**
- 但在详情页上下文中，应该使用**主表主键 id**
- 正确的模板变量应该是 `${id}`

### 核心问题

**详情页无数据的根本原因:**
1. `relationConfig` 和 `queryConfig` 双重过滤机制冲突
2. 关联字段配置错误 (`fbillNo` vs `fid`)
3. 模板变量引用错误 (`${fbill_no}` vs `${id}`)

1. 后端接口未正确实现
2. 字典数据加载失败

## 📝 经验教训

### ERP 详情页签配置规范

1. **relationConfig 与 queryConfig 二选一**
   - 优先使用 `relationConfig` (简单场景)
   - 复杂查询使用 `queryConfig`
   - **禁止同时启用导致冲突**

2. **关联字段必须匹配**
   - `masterField`: 主表的主键或业务主键
   - `detailField`: 明细表的外键字段
   - 两者必须形成 FK-PK 关系

3. **模板变量命名规则**
   - `${字段名}` 引用主表当前行的字段值
   - 确保字段名与主表配置一致

4. **配置前检查表结构**
   - 确认主表和明细表的关联字段
   - 确认字段名的大小写和拼写
