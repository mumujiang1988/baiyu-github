## ⚠️ 二、发现的问题

### 问题 1: 订单状态 (orderStatus) - 使用静态字典

**现象**:
- 前端配置：`orderStatus` 使用静态字典
- 数据库查询：**未找到** `orderStatus` 字典类型
- 实际数据：存储在 `sys_dict_data.f_document_status` 中

**配置内容**:
```json
{
  "type": "static",
  "data": [
    { "label": "未关闭", "value": "A", "type": "success" },
    { "label": "已关闭", "value": "B", "type": "info" },
    { "label": "业务终止", "value": "C", "type": "danger" }
  ],
  "_note": "注意：数据库中没有 orderStatus 字典类型，这是前端硬编码的静态字典"
}
```

**建议**:
- ✅ 保持现状：如果后端 `t_sale_order.orderStatus` 字段确实使用 A/B/C 值
- 🔧 修改方案：如果需要与数据库一致，应该改为查询 `f_document_status`

### 问题 2: 单据状态 (documentStatus) - 配置正确 ✓

**现象**:
- 前端配置：使用静态字典
- 数据库：`sys_dict_data.f_document_status` 有 5 条记录
- 映射关系：**完全匹配** ✓

**配置内容**:
```json
{
  "type": "static",
  "data": [
    { "label": "暂存", "value": "Z", "type": "info" },
    { "label": "创建", "value": "A", "type": "success" },
    { "label": "审核中", "value": "B", "type": "warning" },
    { "label": "已审核", "value": "C", "type": "success" },
    { "label": "重新审核", "value": "D", "type": "primary" }
  ]
}
```

**数据库验证**:
```sql
SELECT dict_label, dict_value FROM sys_dict_data 
WHERE dict_type = 'f_document_status' 
ORDER BY dict_sort;
-- 返回 5 条记录，与前端配置完全一致
```

### 问题 3: 销售员 (salespersons) - 动态加载

**现象**:
- 前端配置：动态字典，从 `sys_user` 表加载
- 映射字段：`nickName` -> `fseller` / `user_id`
- 这不是字典表，是用户表

**配置内容**:
```json
{
  "type": "dynamic",
  "table": "sys_user",
  "config": {
    "api": "/erp/engine/dict/union/salespersons",
    "ttl": 600000,
    "labelField": "nickName",
    "valueField": "fseller",
    "useGlobalCache": true
  },
  "orderBy": [{"field": "nick_name", "direction": "ASC"}],
  "conditions": [{"field": "deleted", "operator": "isNull"}],
  "fieldMapping": {
    "labelField": "nick_name",
    "valueField": "user_id"
  }
}
```

**注意**: 需要确保后端 `/erp/engine/dict/union/salespersons` 接口正确实现
