## 🔧 核心原则

### 前后端字段直连模式（v5.0.0+）

**四大原则**:
1.  **以数据库字段名为准** - 前后端都直接使用数据库字段名
2.  **零映射** - 不需要字段映射配置
3.  **零转换** - 不需要大小写转换或格式转换
4.  **零硬编码** - 不在代码中硬编码字段名，从配置动态读取

**示例说明**:
```javascript
//  正确做法：前后端都使用数据库字段名
// 数据库：t_sale_order_entry.fplanmaterialid (varchar)
// 前端：tableConfig.columns[].prop = "fplanmaterialid"
// 后端：queryWrapper.eq("fplanmaterialid", value)
// 响应：response.rows[].fplanmaterialid = "M001"

// ❌ 错误做法：进行字段映射或转换
// 数据库：fplanmaterialid
// 前端：planMaterialId (驼峰转换)
// 后端：@JsonProperty("planMaterialId") String fplanmaterialid
```

##  注意事项

### 1. 数据库字段命名规范

**主表（t_sale_order）**: 混合命名
- 大部分字段：大写字母开头（FBillNo, FDate, FDocumentStatus）
- 部分字段：下划线分隔（orderStatus, F_tcbl, F_sfbg）

**明细表（t_sale_order_entry）**: 全小写
- 所有字段：全小写，无分隔符（fplanmaterialid, fqty, fprice）

**成本表（t_sale_order_cost）**: 大写 + 下划线
- 主键和大字段：大写字母开头（FID, FBillNo, FBillAllAmount）
- 费用字段：大写 + 下划线（F_hyf, F_bxf, F_gwyhfy）

### 2. 前端配置必须严格匹配

```json
//  正确：严格按照 DESC 命令的输出结果配置
{
  "prop": "fplanmaterialid",    // 不是 fPlanMaterialId
  "prop": "F_hyf",              // 不是 fHyf 或 F_Hyf
  "prop": "FBillAllAmount"      // 不是 fBillAllAmount 或 Fbillallamount
}
```
