### 2.4 创建 Service 服务

创建文件：`k3/service/ErpPageConfigService.java`

> 💡 **提示**: 完整代码参考 `ERP 公共配置表优化方案（完整版）2026-03-22.md` 的 2.3 节

**简化版（仅核心功能）**：

```java
package com.ruoyi.business.k3.service;

import com.alibaba.fastjson2.JSON;
import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.ruoyi.business.k3.domain.ErpPageConfig;
import com.ruoyi.business.k3.domain.ErpPageConfigHistory;
import com.ruoyi.business.k3.mapper.ErpPageConfigMapper;
import jakarta.annotation.Resource;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

@Slf4j
@RequiredArgsConstructor
@Service
public class ErpPageConfigService {
    
    private final ErpPageConfigMapper configMapper;
    
    public String getConfigByModule(String moduleCode) {
        ErpPageConfig config = configMapper.selectByModuleCode(moduleCode);
        if (config == null || !"1".equals(config.getStatus())) {
            return null;
        }
        return config.getConfigContent();
    }
    
    @Transactional(rollbackFor = Exception.class)
    public void saveConfig(ErpPageConfig config, String changeReason) {
        // 验证 JSON 格式
        try {
            JSON.parseObject(config.getConfigContent());
        } catch (Exception e) {
            throw new IllegalArgumentException("配置必须是有效的 JSON 格式");
        }
        
        if (config.getConfigId() != null) {
            // 更新
            ErpPageConfig old = configMapper.selectById(config.getConfigId());
            config.setVersion(old.getVersion() + 1);
            configMapper.updateById(config);
            log.info("更新配置成功：{} v{} -> v{}", 
                     old.getModuleCode(), old.getVersion(), config.getVersion());
        } else {
            // 新增
            config.setVersion(1);
            configMapper.insert(config);
            log.info("新增配置成功：{}", config.getModuleCode());
        }
    }
}
```

### 2.5 创建 Controller

创建文件：`k3/controller/ErpPageConfigController.java`

```java
package com.ruoyi.business.k3.controller;

import cn.dev33.satoken.annotation.SaCheckPermission;
import com.ruoyi.business.k3.domain.ErpPageConfig;
import com.ruoyi.business.k3.service.ErpPageConfigService;
import com.ruoyi.common.core.domain.R;
import com.ruoyi.common.web.core.BaseController;
import lombok.RequiredArgsConstructor;
import org.springframework.web.bind.annotation.*;

@RequiredArgsConstructor
@RestController
@RequestMapping("/erp/config")
public class ErpPageConfigController extends BaseController {
    
    private final ErpPageConfigService configService;
    
    @GetMapping("/{moduleCode}")
    public R<String> getConfig(@PathVariable String moduleCode) {
        String config = configService.getConfigByModule(moduleCode);
        if (config == null) {
            return R.fail("未找到配置：" + moduleCode);
        }
        return R.ok(config);
    }
    
    @SaCheckPermission("erp:config:edit")
    @PostMapping("/save")
    public R<Void> saveConfig(@RequestBody ErpPageConfig config) {
        configService.saveConfig(config, "手动保存");
        return R.ok();
    }
}
```

```vue
<script setup>
import { getPageConfig } from '@/api/erp/config'
import ErpConfigParser from '@/utils/erpConfigParser'
import { ref, onMounted } from 'vue'

const configLoading = ref(false)
const configError = ref(null)
const parsedConfig = ref(null)

onMounted(async () => {
  await loadPageConfig()
})

async function loadPageConfig() {
  configLoading.value = true
  configError.value = null
  
  try {
    const res = await getPageConfig('saleOrder')
    const config = typeof res.data === 'string' ? JSON.parse(res.data) : res.data
    
    const parser = new ErpConfigParser(config)
    parsedConfig.value = parser.parseAll()
    
  } catch (error) {
    console.error('配置加载失败:', error)
    configError.value = `加载配置失败：${error.message}`
    
  } finally {
    configLoading.value = false
  }
}
</script>

<template>
  <div class="app-container">
    <!-- 加载中 -->
    <el-skeleton :rows="10" animated v-if="configLoading" />
    
    <!-- 加载失败 -->
    <el-empty 
      v-else-if="configError" 
      :description="configError"
    >
      <el-button type="primary" @click="loadPageConfig">重新加载</el-button>
    </el-empty>
    
    <!-- 正常渲染 -->
    <template v-else-if="parsedConfig">
      <!-- ... 现有代码 ... -->
    </template>
  </div>
</template>
```

## 🔍 故障排查

### 问题 1: 配置加载失败

**错误信息**：`未找到配置：saleOrder`

**解决方案**：
```sql
-- 检查配置是否存在
SELECT * FROM erp_page_config WHERE module_code = 'saleOrder';

-- 如果不存在，重新插入
INSERT INTO erp_page_config (...) VALUES (...);
```

### 问题 2: 前端跨域错误

**错误信息**：`Access to XMLHttpRequest at ... has been blocked by CORS policy`

**解决方案**：
编辑 `By-web/vite.config.js`：
```javascript
server: {
  proxy: {
    '/erp': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

### 问题 3: JSON 解析失败

**错误信息**：`Unexpected token U in JSON at position 0`

**原因**：返回的是 Undefined 或 HTML 错误页面

**解决方案**：
```javascript
// 添加错误处理
try {
  const config = typeof res.data === 'string' 
    ? JSON.parse(res.data) 
    : res.data
} catch (error) {
  console.error('JSON 解析失败:', res.data)
}
```
