/**
 * 获取国家地区列表

@GetMapping("/nation")
public Result getNationList() {
    List<Country> list = countryMapper.selectList();
    // 转换为前端需要的格式

### 2. **SysDictData (系统字典数据)** 

#### Controller 接口
**文件**: `SysDictDataController.java` (位于 ruoyi-system 模块)

**路径**: `/system/dict/data`

**接口**:
```java
// 根据字典类型查询字典数据信息 (第 74 行)
@GetMapping(value = "/type/{dictType}")
public R<List<SysDictDataVo>> dictType(@PathVariable String dictType) {
    List<SysDictDataVo> data = dictTypeService.selectDictDataByType(dictType);
    if (ObjectUtil.isNull(data)) {
        data = new ArrayList<>();
    }
    return R.ok(data);
}
```

**其他接口**:
- `GET /list` - 分页查询字典数据列表
- `GET /{dictCode}` - 查询字典数据详细
- `POST /` - 新增字典数据
- `PUT /` - 修改字典数据
- `DELETE /{dictCodes}` - 删除字典数据

#### 权限要求
```java
@SaCheckPermission("system:dict:list")      // 查询
@SaCheckPermission("system:dict:query")     // 详情
@SaCheckPermission("system:dict:add")       // 新增
@SaCheckPermission("system:dict:edit")      // 修改
@SaCheckPermission("system:dict:remove")    // 删除
@SaCheckPermission("system:dict:export")    // 导出
```

####  可用结论
**这个接口可以直接使用!** 

**前端配置示例**:
```json
"dictionaryConfig": {
  "someDict": {
    "api": "/system/dict/data/type/{dictType}",

    "valueField": "dictValue"
  }
}
```

## 🎯 最佳实践建议

### 1. **分类管理策略**

| 字典类型 | 推荐存储表 | 理由 |
|---------|-----------|------|
| **系统级字典** | `sys_dict_data` | 已有完善管理功能 |
| **业务级字典** | `dictionary_table` | 灵活，可自定义 |
| **基础数据** | 专用表 (如 country) | 数据结构复杂，需专用表 |

### 2. **具体实施**

#### A. 结算币别 (currency)
- **选项 1**: 使用 `sys_dict_data`, dict_type = `currency_type`
- **选项 2**: 使用 `dictionary_table`, dict_name = `currency`

#### B. 收款条件 (payment_term)
- **选项 1**: 使用 `sys_dict_data`, dict_type = `payment_term`
- **选项 2**: 使用 `dictionary_table`, dict_name = `payment_term`

#### C. 国家地区 (nation)
- **推荐**: 使用 `country` 专用表
- **理由**: 国家数据相对固定，且有更多属性 (ID 代码、中文名、英文名等)
