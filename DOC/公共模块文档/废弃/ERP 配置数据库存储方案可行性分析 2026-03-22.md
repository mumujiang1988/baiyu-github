## 一、方案对比

### 📊 当前方案 vs 数据库方案

| 维度 | 当前 JSON 文件方案 | 数据库存储方案 | 优势 |
|------|------------------|--------------|------|
| **读取方式** | 静态 import 或 fetch | API 动态加载 | ✅ 实时更新 |
| **修改流程** | 修改文件 → 重新部署 | 后台页面修改 → 立即生效 | ✅ 无需重启 |
| **版本管理** | Git 版本控制 | 可增加版本号字段 | ✅ 灵活追溯 |
| **权限控制** | 文件系统权限 | 细粒度 RBAC 权限 | ✅ 精确控制 |
| **多环境同步** | 手动复制文件 | 共享数据库或数据同步 | ✅ 自动化 |
| **配置大小** | 113KB（无限制） | VARCHAR(500) 限制 ⚠️ | ❌ 需扩展字段 |
| **查询性能** | 快（内存缓存） | 中等（需缓存优化） | ⚠️ 需缓存策略 |
| **调试难度** | 易（直接查看文件） | 中（需查询数据库） | ⚠️ 需管理工具 |

## 五、性能优化策略

### 🚀 缓存策略

#### 1. **多级缓存架构**
```
浏览器 LocalStorage
      ↓↑
Redis 缓存（TTL: 1 小时）
      ↓↑
MySQL 数据库
```

#### 2. **前端缓存实现**
```javascript
// utils/cache.js
const CACHE_PREFIX = 'erp:config:';
const CACHE_TTL = 3600000; // 1 小时

export function getCachedConfig(moduleCode) {
  const cached = localStorage.getItem(CACHE_PREFIX + moduleCode);
  if (cached) {
    const { data, timestamp } = JSON.parse(cached);
    if (Date.now() - timestamp < CACHE_TTL) {
      return data;
    }
  }
  return null;
}

export function cacheConfig(moduleCode, config) {
  localStorage.setItem(CACHE_PREFIX + moduleCode, JSON.stringify({
    data: config,
    timestamp: Date.now()
  }));
}

// Vue 组件中使用
async loadPageConfig() {
  // 先尝试本地缓存
  const cached = getCachedConfig('saleOrder');
  if (cached) {
    this.parsedConfig = cached;
    return;
  }
  
  // 调用 API
  const res = await getPageConfig('saleOrder');
  const config = JSON.parse(res.data);
  
  // 缓存到本地
  cacheConfig('saleOrder', config);
  
  // 解析并渲染
  const parser = new ErpConfigParser(config);
  this.parsedConfig = parser.parseAll();
}
```

#### 3. **Redis 缓存策略**
```java
@Service
public class ErpPageConfigService {
    
    /**
     * 获取配置（两级缓存）
     */
    public String getConfigByModule(String moduleCode) {
        // 1. 先查 Redis
        String config = redisUtils.getCacheObject("erp:page:config:" + moduleCode);
        if (StringUtils.isNotBlank(config)) {
            return config;
        }
        
        // 2. 查询数据库
        ErpPageConfig dbConfig = baseMapper.selectOne(
            new LambdaQueryWrapper<ErpPageConfig>()
                .eq(ErpPageConfig::getModuleCode, moduleCode)
        );
        
        if (dbConfig != null) {
            // 3. 写入 Redis（1 小时过期）
            redisUtils.setCacheObject(
                "erp:page:config:" + moduleCode,
                dbConfig.getConfigContent(),
                1, TimeUnit.HOURS
            );
            return dbConfig.getConfigContent();
        }
        
        return null;
    }
    
    /**
     * 更新配置（清除所有缓存）
     */
    public void updateConfig(String moduleCode, String content) {
        // 更新数据库
        ErpPageConfig config = new ErpPageConfig();
        config.setModuleCode(moduleCode);
        config.setConfigContent(content);
        baseMapper.updateById(config);
        
        // 清除 Redis 缓存
        String cacheKey = "erp:page:config:" + moduleCode;
        redisUtils.deleteObject(cacheKey);
        
        // 记录版本历史（可选）
        saveConfigHistory(config);
    }
}
```

## 七、版本控制和回滚

### 📝 版本管理策略

#### 1. **自动版本号递增**
```java
@Transactional(rollbackFor = Exception.class)
public void updateConfig(String moduleCode, String content, String reason) {
    ErpPageConfig config = baseMapper.selectOne(
        new LambdaQueryWrapper<ErpPageConfig>()
            .eq(ErpPageConfig::getModuleCode, moduleCode)
    );
    
    if (config != null) {
        // 递增版本号
        config.setVersion(config.getVersion() + 1);
        config.setConfigContent(content);
        baseMapper.updateById(config);
        
        // 保存历史记录
        ErpConfigHistory history = new ErpConfigHistory();
        history.setConfigId(config.getConfigId());
        history.setVersion(config.getVersion());
        history.setConfigContent(content);
        history.setChangeReason(reason);
        historyMapper.insert(history);
    }
}
```

#### 2. **版本回滚功能**
```java
/**
 * 回滚到指定版本
 */
public void rollbackToVersion(Long configId, Integer targetVersion) {
    // 查找目标版本的历史记录
    ErpConfigHistory history = historyMapper.selectOne(
        new LambdaQueryWrapper<ErpConfigHistory>()
            .eq(ErpConfigHistory::getConfigId, configId)
            .eq(ErpConfigHistory::getVersion, targetVersion)
    );
    
    if (history == null) {
        throw new ServiceException("指定的版本不存在");
    }
    
    // 更新当前配置
    ErpPageConfig config = baseMapper.selectById(configId);
    config.setConfigContent(history.getConfigContent());
    config.setVersion(config.getVersion() + 1);
    baseMapper.updateById(config);
}
```

### ✅ 核心优势

1. **实时生效** - 无需重新部署，修改后立即生效
2. **集中管理** - 统一后台管理所有 ERP 模块配置

4. **权限管控** - 细粒度的读写权限控制

3. **验证机制** - 必须有 JSON Schema 验证防止错误配置
4. **备份机制** - 建议定期导出 SQL 备份

- ✅ 风险可控
