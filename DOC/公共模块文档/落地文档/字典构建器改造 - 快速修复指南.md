# 字典构建器改造 - 快速修复指南

## 🚀 问题现象

浏览器控制台出现警告：
```
 字典未注册：salespersons
 字典未注册：orderStatus
```

## 🔧 原因分析

Redis 缓存中的配置数据损坏或过期，导致字典构建器引擎无法正确初始化。

##  解决方案（3 步搞定）

### 步骤 1：清理 Redis 缓存

打开 PowerShell，执行：

```powershell
# 连接到 Redis（需要密码）
redis-cli -h localhost -p 6379 -a difyai123456

# 删除 ERP 配置缓存
DEL "erp_config"
DEL "{erp_config}:redisson_options"

# 验证是否删除成功
KEYS "*erp*"

# 退出
exit
```

**或者使用一行命令**：
```powershell
redis-cli -h localhost -p 6379 -a difyai123456 FLUSHDB
```

>  **注意**: `FLUSHDB` 会清空整个数据库，请谨慎使用！

---

### 步骤 2：刷新浏览器

1. **强制刷新页面**：按 `Ctrl + F5`
2. **清除浏览器缓存**：按 `Ctrl + Shift + Delete`
3. **重新访问销售订单页面**：http://localhost:8899/business/saleorder

---

### 步骤 3：验证结果

####  成功的标志

浏览器控制台应该显示：
```
 字典构建器预加载完成
 字典构建器预加载完成
```

Network 面板应该有以下请求：
```
 GET /erp/engine/dictionary/salespersons/data?moduleCode=saleorder
 GET /erp/engine/dictionary/orderStatus/data?moduleCode=saleorder
```

## 📊 预防措施

### 1. 定期清理缓存

添加定时任务（每周执行一次）：

```powershell
# 添加到 Windows 任务计划程序
$action = New-ScheduledTaskAction -Execute "redis-cli" -Argument "-h localhost -p 6379 -a difyai123456 DEL `"erp_config`""
$trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Monday -At 2am
Register-ScheduledTask -TaskName "ERP Dictionary Cache Cleanup" -Action $action -Trigger $trigger -RunLevel Highest
```

### 2. 增加缓存健康检查

在后端添加健康检查接口：

```java
@GetMapping("/erp/cache/health")
public R<?> checkCacheHealth() {
    try {
        // 检查 Redis 连接
        redisTemplate.opsForValue().set("health_check", "ok", 10, TimeUnit.SECONDS);
        String value = redisTemplate.opsForValue().get("health_check");
        
        if ("ok".equals(value)) {
            return R.ok("缓存健康");
        } else {
            return R.fail("缓存异常");
        }
    } catch (Exception e) {
        return R.fail("缓存连接失败：" + e.getMessage());
    }
}
```

### 3. 监控告警

添加字典加载失败次数监控：

```javascript
// 在前端添加监控
let dictLoadErrorCount = 0

const originalWarn = console.warn
console.warn = function(...args) {
  if (args[0] && args[0].includes('字典未注册')) {
    dictLoadErrorCount++
    
    // 超过 3 次触发告警
    if (dictLoadErrorCount > 3) {
      ElMessage.error('字典加载失败次数过多，请清理缓存')
      dictLoadErrorCount = 0
    }
  }
  originalWarn.apply(console, args)
}
```
