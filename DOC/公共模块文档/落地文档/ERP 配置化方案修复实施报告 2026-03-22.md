##  修复技术细节

### 1. SQL注入防护原理

**攻击场景**:
```javascript
// 恶意请求
{
  "field": "id) OR 1=1 --",
  "searchType": "like",
  "value": "test"
}
```

**防护机制**:
```java
// 白名单校验
if (!ALLOWED_FIELDS.contains(field)) {
    log.warn("非法字段访问尝试: {}", field);
    continue; // 拒绝执行
}
```

**效果**: 恶意字段被拒绝,无法注入SQL

---

### 2. 代码注入防护原理

**攻击场景**:
```json
{
  "condition": "java.lang.Runtime.getRuntime().exec('rm -rf /')",
  "billData": {}
}
```

**防护机制**:
```java
// 移除ScriptEngine,使用安全的表达式解析器
// 只支持预定义的运算符和比较操作
private boolean evaluateComparison(...) {
    // 只处理数值和字符串比较
    // 无法执行任意代码
}
```

**效果**: 恶意代码无法执行
