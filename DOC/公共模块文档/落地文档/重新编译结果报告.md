### 根本原因

这些错误都是 **MapStruct 自动生成的代码读取失败**，与本次 ERP 文件清理**完全无关**。

**错误特点**：
1. ❌ 都是 `target/generated-sources/annotations/` 目录下的自动生成文件
2. ❌ 都是 MapStruct 的 Mapper 实现类（`*MapperImpl.java`）
3. ❌ 都是业务模块（`com.ruoyi.business.*`），不是系统管理模块
4. ❌ 错误类型是"读取文件时出错"，不是编译语法错误

### 可能的原因

1. **文件被占用** - Windows 环境下常见
2. **IDE 正在索引** - IDE 锁定了文件
3. **杀毒软件干扰** - 实时扫描导致文件锁定
4. **Maven 并发问题** - 多线程编译导致竞争条件

### 解决方案

#### 方案 1：清理 target 并重新编译（推荐）

```powershell
cd d:\baiyuyunma\baiyu-github\baiyu-github\baiyu-ruoyi
mvn clean package -DskipTests -pl ruoyi-admin-wms -am
```

#### 方案 2：停止 IDE 和后台进程

1. 关闭 IntelliJ IDEA / Eclipse
2. 停止所有 Java 进程
   ```powershell
   taskkill /F /IM java.exe
   ```
3. 手动删除 target 目录
   ```powershell
   Remove-Item -Recurse -Force ruoyi-admin-wms\target
   ```
4. 重新编译
   ```powershell
   mvn compile -pl ruoyi-admin-wms -am
   ```

#### 方案 3：禁用 MapStruct 注解处理（临时方案）

```xml
<!-- pom.xml 中添加 -->
<properties>
    <maven.compiler.proc>none</maven.compiler.proc>
</properties>
```
