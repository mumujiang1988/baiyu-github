```
系统菜单 → ERP 管理 → 配置管理
```

### 页面路由

| 页面 | 路由 | 说明 |
|------|------|------|
| 配置列表 | `/erp/config` | 主页面，管理所有配置 |
| 新增配置 | `/erp/config/editor` | 新增配置页面 |

| 历史版本 | `/erp/config/history?id={id}` | 查看配置历史 |

## 📖 功能说明

-  一键回滚到任意版本

## 📝 操作指南

### 一、新增配置

**步骤**:

1. 点击「新增配置」按钮
2. 填写基本信息：
   - **模块编码**: 如 `saleOrder`, `purchaseOrder`（小驼峰命名）
   - **配置名称**: 如「销售订单页面配置」
   - **配置类型**: 选择 PAGE/DICT/PUSH/APPROVAL 等
   - **是否公共**: 决定其他用户是否可见
3. 编辑配置内容：
   - 方式 1: 直接编写 JSON
   - 方式 2: 点击「加载模板」选择预设模板
   - 方式 3: 从其他配置复制后修改
4. 点击「验证 JSON」确保格式正确
5. 点击「格式化」美化 JSON 格式
6. （可选）填写变更原因说明
7. 点击「保存配置」

**示例配置内容**:

```json
{
  "pageConfig": {
    "title": "销售订单",
    "permissionPrefix": "k3:sale",
    "primaryKey": "id"
  },
  "apiConfig": {
    "methods": {
      "list": "/k3/sale/list",
      "get": "/k3/sale/{id}",
      "add": "/k3/sale/add",
      "edit": "/k3/sale/edit",
      "delete": "/k3/sale/{id}"
    }
  },
  "searchConfig": {
    "showSearch": true,
    "defaultExpand": true,
    "fields": [
      {
        "field": "billNo",
        "label": "单据编号",
        "type": "input",
        "searchType": "like"
      }
    ]
  }
}
```

---

### 二、编辑配置

**步骤**:

1. 在配置列表中找到目标配置
2. 点击「编辑」按钮
3. 修改配置内容
4. 点击「验证 JSON」确保格式正确
5. 填写变更原因（建议填写，便于追溯）
6. 点击「保存配置」

**注意事项**:
-  模块编码和配置类型不可修改
-  每次修改都会生成新版本
-  建议填写变更原因

---

### 三、查看配置

**步骤**:

1. 在配置列表中点击某行的「查看」按钮
2. 查看配置基本信息和内容
3. 可点击「编辑配置」进入编辑模式

---

### 四、查看历史版本

**步骤**:

1. 在配置列表中点击某行的「历史」按钮
2. 查看该配置的所有历史版本
3. 可执行以下操作：
   - **查看**: 查看某个版本的详细内容
   - **对比**: 对比两个版本的差异
   - **回滚**: 回滚到指定版本

**版本对比模式**:
- **并排对比**: 左右同时显示两个版本
- **差异对比**: 显示差异内容（- 删除，+ 新增）

---

### 五、回滚配置

**场景**: 当发现新配置有问题时，可快速回滚到之前的稳定版本

**步骤**:

1. 进入配置历史版本页面
2. 找到要回滚到的目标版本
3. 点击该版本的「回滚」按钮
4. 确认回滚操作
5. 系统自动生成新版本（回滚版本）

**注意事项**:
-  回滚会生成新版本，不会删除已有版本
-  回滚后立即生效，请谨慎操作

---

### 六、复制配置

**场景**: 基于现有配置快速创建相似配置

**步骤**:

1. 在配置列表中找到源配置
2. 点击「更多」→「复制配置」
3. 修改模块编码和配置名称
4. 调整配置内容
5. 保存为新配置

---

### 七、导出/导入配置

**导出场景**: 备份配置或迁移到其他环境

**导出步骤**:
1. 在配置列表中找到目标配置
2. 点击「更多」→「导出配置」
3. 下载 JSON 文件

**导入步骤**:
1. 点击「新增配置」
2. 手动粘贴 JSON 内容或从文件读取
3. 填写其他必填信息
4. 保存配置

## 💡 最佳实践

### 配置命名规范

```
 推荐:
- 模块编码：saleOrder, purchaseOrder, inventoryCheck
- 配置名称：销售订单页面配置、采购入库单审批配置

 不推荐:
- 模块编码：Sale_Order, purchase-order, test123
- 配置名称：测试配置、aaa 配置
```

### 配置内容组织

**推荐的 JSON 结构**:

```json
{
  "pageConfig": { ... },      // 页面基础配置
  "apiConfig": { ... },       // API 接口配置
  "searchConfig": { ... },    // 搜索区域配置
  "tableConfig": { ... },     // 表格列表配置
  "formConfig": { ... },      // 表单弹窗配置
  "approvalConfig": { ... }   // 审批流程配置（可选）
}
```

### 版本管理建议

1. **小步快跑**: 每次只修改少量内容，频繁保存
2. **详细记录**: 认真填写变更原因，便于后续追溯
3. **及时回滚**: 发现问题立即回滚到稳定版本
4. **定期清理**: 删除废弃的测试配置

### 权限控制

```
建议权限分配:
- 配置查看：所有用户（公共配置）
- 配置编辑：实施顾问、开发人员
- 配置删除：系统管理员
- 版本回滚：项目负责人
```

## ❓ 常见问题

### Q1: 配置保存后为什么没有立即生效？

**A**: 配置保存后通常会缓存 5 分钟，可通过以下方式立即生效：
- 清除浏览器缓存
- 等待 5 分钟自动刷新
- 联系管理员清除服务端缓存

---

### Q2: JSON 格式验证失败怎么办？

**A**: 
1. 检查是否有语法错误（缺少逗号、引号等）
2. 点击「格式化」按钮，系统会自动纠正部分错误
3. 使用在线 JSON 验证工具检查
4. 从模板重新加载

---

### Q3: 误删了配置怎么办？

**A**: 
1. 如果是刚删除，立即联系管理员从数据库恢复
2. 如果有导出文件，重新导入即可
3. 如果有历史版本，从其他环境复制

---

### Q4: 如何批量修改多个配置？

**A**: 
当前版本暂不支持批量修改，建议：
1. 逐个修改配置
2. 导出后批量修改 JSON 文件
3. 删除旧配置后批量导入新配置

---

### Q5: 配置内容太多，编辑起来很卡怎么办？

**A**: 
1. 将配置拆分为多个子配置
2. 使用外部编辑器编辑后粘贴进来
3. 优化配置结构，移除冗余内容
