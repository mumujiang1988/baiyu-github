## 🎯 功能汇总

### **一、动态查询引擎**

#### 功能描述
基于配置动态构建查询条件，支持多种查询操作符和字段映射。

#### API 方法 (3 个)

| 方法名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `executeDynamicQuery` | 执行动态查询 | moduleCode, queryParams, searchConfig | QueryWrapper |
| `buildQueryConditions` | 构建查询条件 | searchConfig, queryParams | QueryWrapper |
| `getAvailableQueryTypes` | 获取查询类型 | 无 | List<String> |

#### 支持的查询类型
```javascript
['like', 'left_like', 'right_like', 'in', 'between', 
 'gt', 'ge', 'lt', 'le', 'ne', 'eq']
```

#### 使用示例
```javascript
import { executeDynamicQuery } from '@/api/erp/engine/query'

const result = await executeDynamicQuery({
  moduleCode: 'saleOrder',
  queryParams: { fbillNo: 'SO20260322001' },
  searchConfig: {
    fields: [
      { field: 'fbillNo', type: 'like' },
      { field: 'fdate', type: 'between' }
    ]
  }
})
```

### **二、表单验证引擎**

#### 功能描述
基于配置执行表单数据验证，支持多种验证规则和自定义验证器。

#### API 方法 (4 个)

| 方法名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `executeValidation` | 执行表单验证 | moduleCode, formData, validationConfig | ValidationResult |
| `batchValidate` | 批量验证 | formDataList, validationConfig | List<ValidationResult> |
| `getAvailableValidationRules` | 获取验证规则 | 无 | List<String> |
| `validateField` | 验证单个字段 | field, value, rule | FieldResult |

#### 支持的验证规则
```javascript
['required', 'email', 'phone', 'number', 'integer',
 'min', 'max', 'minLength', 'maxLength', 'pattern', 'range']
```

#### 使用示例
```javascript
import { executeValidation } from '@/api/erp/engine/validation'

const result = await executeValidation({
  moduleCode: 'saleOrder',
  formData: {
    fBillNo: 'SO20260322001',
    fAmount: 10000
  },
  validationConfig: {
    rules: {
      fBillNo: [{ required: true, message: '单据号必填' }],
      fAmount: [{ type: 'number', min: 0 }]
    }
  }
})
```

### **三、审批流程引擎**

#### 功能描述
基于配置的审批流程，自动判断当前审批步骤、执行审批操作和权限检查。

#### API 方法 (7 个)

| 方法名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `getCurrentApprovalStep` | 获取当前步骤 | moduleCode, billId, billData | StepInfo |
| `executeApproval` | 执行审批 | moduleCode, billId, action, opinion | ApprovalResult |
| `checkApprovalPermission` | 检查权限 | billId, moduleCode, userId, userRoles | PermissionResult |
| `getApprovalHistory` | 获取历史 | billId, moduleCode | List<HistoryItem> |
| `getWorkflowDefinition` | 获取流程定义 | moduleCode | WorkflowDef |
| `transferApproval` | 转审操作 | billId, targetUserId | TransferResult |
| `withdrawApproval` | 撤回审批 | billId, moduleCode | WithdrawResult |

#### 审批动作
```javascript
['AUDIT', 'APPROVE', 'REJECT', 'WITHDRAW', 'TRANSFER']
```

#### 使用示例
```javascript
import { 
  getCurrentApprovalStep,
  executeApproval 
} from '@/api/erp/engine/approval'

// 获取当前步骤
const step = await getCurrentApprovalStep({
  moduleCode: 'saleOrder',
  billId: '123456',
  billData: orderData
})

// 执行审批
await executeApproval({
  moduleCode: 'saleOrder',
  billId: '123456',
  action: 'APPROVE',
  opinion: '同意此订单'
})
```

### **四、下推引擎**

#### 功能描述
基于配置的映射关系，将源单据数据转换为下游单据数据。

#### API 方法 (8 个)

| 方法名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `getPushTargets` | 获取可下推目标 | moduleCode | List<TargetInfo> |
| `executePushDown` | 执行下推 | sourceId, sourceModule, targetModule, confirmData | PushResult |
| `previewPushDown` | 预览下推 | sourceId, sourceModule, targetModule | PreviewData |
| `batchPushDown` | 批量下推 | sourceIds, sourceModule, targetModule | BatchResult |
| `getPushMappingConfig` | 获取映射配置 | sourceModule, targetModule | MappingConfig |
| `validatePushData` | 验证下推数据 | sourceData, mappingConfig | ValidateResult |
| `cancelPushDown` | 取消下推 | targetId, targetModule | CancelResult |
| `getPushHistory` | 获取下推历史 | billId, moduleCode | List<HistoryItem> |

#### 使用示例
```javascript
import { 
  getPushTargets,
  executePushDown 
} from '@/api/erp/engine/push'

// 获取可下推目标
const targets = await getPushTargets('saleOrder')
// 返回：[
//   { moduleCode: 'deliveryOrder', moduleName: '发货通知单' },
//   { moduleCode: 'outboundOrder', moduleName: '出库单' }
// ]

// 执行下推
await executePushDown({
  sourceId: '123456',
  sourceModule: 'saleOrder',
  targetModule: 'deliveryOrder',
  confirmData: { 
    fDeliveryType: 'express',
    fAddress: '北京市朝阳区'
  }
})
```

## 🔗 配置化前后端集成

### **前端调用流程**

```
用户操作
  ↓
BusinessConfigurable.vue 组件
  ↓
调用 getApiMethod(methodKey)
  ↓
动态导入对应的 API 方法
  ↓
调用引擎 API (query/validation/approval/push)
  ↓
发送 HTTP 请求到后端
```

### **后端处理流程**

```
HTTP 请求
  ↓
ErpEngineController (路由层)
  ↓
Engine Service (业务逻辑层)
  ↓
Mapper/Database (数据持久层)
  ↓
返回结果
```

### **典型集成示例**

#### 1. **查询列表**

```javascript
// 前端：BusinessConfigurable.vue
const handleQuery = async () => {
  const apiMethod = await getApiMethod('list')
  if (!apiMethod) return
  
  loading.value = true
  try {
    const response = await apiMethod(queryParams.value)
    if (response.code === 200) {
      tableData.value = response.rows
      total.value = response.total
    }
  } finally {
    loading.value = false
  }
}
```

#### 2. **执行审批**

```javascript
// 前端：BusinessConfigurable.vue
const handleAudit = async (row, action) => {
  const apiMethod = await getApiMethod('audit')
  if (!apiMethod) return
  
  try {
    await apiMethod({
      id: row.id,
      action: action,  // 'AUDIT' | 'APPROVE' | 'REJECT'
      opinion: '同意'
    })
    ElMessage.success('操作成功')
    refreshTable()
  } catch (error) {
    ElMessage.error('操作失败：' + error.message)
  }
}
```

#### 3. **下推操作**

```javascript
// 前端：BusinessConfigurable.vue
const handlePush = async (row, targetModule) => {
  const { executePushDown } = await import('@/api/erp/engine/push')
  
  try {
    const result = await executePushDown({
      sourceId: row.fbillNo,
      sourceModule: 'saleOrder',
      targetModule: targetModule,
      confirmData: {}
    })
    
    if (result.success) {
      ElMessage.success('下推成功')
    } else {
      ElMessage.warning('下推失败：' + result.message)
    }
  } catch (error) {
    ElMessage.error('下推异常：' + error.message)
  }
}
```

## 📞 技术支持

### **常见问题**

1. **菜单不显示？**
   - 检查 sys_menu 表是否有数据
   - 清除 Redis 缓存
   - 重新登录

2. **API 调用失败？**
   - 检查后端是否启动
   - 查看控制台错误日志
   - 验证权限标识是否正确

3. **配置不生效？**
   - 检查 JSON 格式是否正确
   - 清除浏览器缓存
   - 刷新页面
