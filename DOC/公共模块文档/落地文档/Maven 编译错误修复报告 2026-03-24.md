**修复时间**: 2026-03-24  
**问题类型**: Maven 多模块项目编译错误 + 类加载异常  
**影响范围**: 后端服务无法启动  

---

## 📋 问题概述

在启动 RuoYi-WMS 项目时遇到一系列 Maven 编译错误和运行时类加载异常，导致后端服务无法正常启动。

### 主要错误现象

1. **编译错误** - 9 个 Service 实现类无法通过编译
2. **类加载错误** - `BaseMapperPlus cannot be resolved to a type`
3. **文件锁定问题** - Maven clean 插件无法删除 target 目录
4. **MapStruct 注解处理器错误** - 生成的 MapperImpl 文件损坏

### 2. Maven 构建问题

#### 问题 1: 文件锁定
```
[ERROR] Failed to execute goal org.apache.maven.plugins:maven-clean-plugin:3.2.0:clean 
Failed to delete D:\...\target\generated-sources\annotations\...
```

**原因**: Java 进程未完全释放文件句柄，导致 Maven 无法清理 target 目录

#### 问题 2: MapStruct 生成的文件损坏
```
[ERROR] 读取 ErpPageConfigHistoryVoToErpPageConfigHistoryMapperImpl.java 时出错
```

**原因**: 增量编译导致 MapStruct 注解处理器生成的文件损坏

#### 问题 3: 依赖模块未安装到本地仓库
```
[ERROR] The import com.ruoyi.common.mybatis cannot be resolved
[ERROR] BaseMapperPlus cannot be resolved to a type
```

**原因**: ruoyi-system、ruoyi-common-mybatis 等模块未执行 `mvn install`,运行时找不到依赖

## 🛠️ 修复方案

### 修复 1: 修正代码编译错误

#### 1.1 修复 SysUserMapper 方法调用

**涉及文件**:
- `SupplierServiceImpl.java`
- `SaleOrderServiceImpl.java` (2 处)

**修复内容**:
```java
// 修改前
SysUser user = sysUserMapper.selectById(userId);

// 修改后
SysUserVo user = sysUserMapper.selectVoById(userId);
```

#### 1.2 修复导入路径

**文件**: `SaleOrderServiceImpl.java`

```java
// 修改前
import com.ruoyi.business.util.SaleOrderCostCalculator;

// 修改后
import com.ruoyi.business.utils.SaleOrderCostCalculator;
```

#### 1.3 修复包声明

**文件**: `SaleOrderCostCalculator.java`

```java

package com.ruoyi.business.util;

// 修改后

```

#### 1.4 修复类型转换

**文件**: `PurchaseOrderServiceImpl.java`

```java

// 修改后
```

```java

```

## 🎯 经验总结

### 关键教训

1. **Java 包声明必须与文件路径一致**
   - 文件在 `utils` 目录，package 必须是 `com.ruoyi.business.utils`

2. **Maven 多模块项目必须先 install**
   - 子模块需要安装到本地仓库才能被其他模块引用
   - 使用 `mvn clean install` 而非 `mvn compile`

3. **文件锁定问题的处理**
   - 先终止所有 Java 进程
   - 再手动删除 target 目录
   - 最后重新编译

4. **MapStruct 注解处理器的特性**
   - 增量编译可能导致生成的文件损坏
   - 遇到问题时执行 `mvn clean` 彻底清理

5. **启动脚本的智能化**
   - 检测依赖是否已安装
   - 检测 JAR 文件新旧程度
   - 避免不必要的重复编译

### 最佳实践

```bash
# 完整的清理编译流程
taskkill /F /IM java.exe
cd project-root
Remove-Item -Recurse -Force target
mvn clean install -DskipTests

# 启动服务
./start-services.ps1
```
