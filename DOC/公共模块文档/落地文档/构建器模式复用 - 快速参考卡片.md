## 🎯 核心架构

```
┌─────────────────────────────────────────┐
│         前端 JSON 配置                    │
│  - tableName                            │
│  - queryConfig                          │
│  - fieldMapping                         │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│   ErpEngineController (统一入口)        │
│  - /erp/engine/query/execute           │
│  - /erp/engine/dictionary/{name}/data  │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│   buildQueryFromBuilderMode()           │
│   解析 queryConfig → QueryWrapper       │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│   SuperDataPermissionServiceImpl        │
│   selectListByModule()                  │
│   selectPageByModuleWithTableName()     │
└──────────────┬──────────────────────────┘
               ↓
┌─────────────────────────────────────────┐
│   JdbcTemplate 执行 SQL                 │
│   生成 SQL → 查询数据库 → 返回结果      │
└─────────────────────────────────────────┘
```

## 📝 表格数据配置模板

### 基础查询

```json
{
  "pageConfig": {
    "tableName": "t_sale_order"
  },
  "queryConfig": {
    "conditions": [
      {
        "field": "FDocumentStatus",
        "operator": "eq",
        "value": "C"
      }
    ],
    "orderBy": [
      {
        "field": "FCreateDate",
        "direction": "DESC"
      }
    ]
  }
}
```

**生成的 SQL**:
```sql
SELECT * FROM t_sale_order 
WHERE FDocumentStatus = 'C' 
ORDER BY FCreateDate DESC
```

---

### 复杂查询

```json
{
  "queryConfig": {
    "conditions": [
      {
        "field": "FDate",
        "operator": "between",
        "value": ["2026-01-01", "2026-12-31"]
      },
      {
        "field": "FAmount",
        "operator": "gt",
        "value": 10000
      },
      {
        "field": "FCustomerName",
        "operator": "like",
        "value": "华为"
      },
      {
        "field": "deleted",
        "operator": "isNull"
      }
    ],
    "orderBy": [
      {
        "field": "FDate",
        "direction": "DESC"
      },
      {
        "field": "FBillNo",
        "direction": "ASC"
      }
    ]
  }
}
```

**生成的 SQL**:
```sql
SELECT * FROM t_sale_order 
WHERE FDate BETWEEN '2026-01-01' AND '2026-12-31'
  AND FAmount > 10000
  AND FCustomerName LIKE '%华为%'
  AND deleted IS NULL
ORDER BY FDate DESC, FBillNo ASC
```

## 🔧 支持的运算符（12 种）

| 运算符 | 说明 | 示例 | SQL 对应 |
|--------|------|------|----------|
| `eq` | 等于 | `FStatus eq 'C'` | `FStatus = 'C'` |
| `ne` | 不等于 | `FStatus ne 'A'` | `FStatus <> 'A'` |
| `gt` | 大于 | `FAmount gt 1000` | `FAmount > 1000` |
| `ge` | 大于等于 | `FDate ge '2026-01-01'` | `FDate >= '2026-01-01'` |
| `lt` | 小于 | `FAmount lt 5000` | `FAmount < 5000` |
| `le` | 小于等于 | `FDate le '2026-12-31'` | `FDate <= '2026-12-31'` |
| `like` | 模糊匹配 | `FCustomerName like '%华为%'` | `FCustomerName LIKE '%华为%'` |
| `left_like` | 左模糊 | `FCustomerName left_like '%公司'` | `FCustomerName LIKE '%公司'` |
| `right_like` | 右模糊 | `FCustomerName right_like '华为%'` | `FCustomerName LIKE '华为%'` |
| `in` | IN 条件 | `FStatus in ['A','B']` | `FStatus IN ('A', 'B')` |
| `between` | BETWEEN | `FDate between ['2026-01-01', '2026-12-31']` | `FDate BETWEEN '2026-01-01' AND '2026-12-31'` |
| `isNull` | IS NULL | `FRemark isNull` | `FRemark IS NULL` |
| `isNotNull` | IS NOT NULL | `FPhone isNotNull` | `FPhone IS NOT NULL` |

## 🚀 多表格并行查询

### 配置结构

```json
{
  "pageConfig": {
    "tableName": "t_sale_order"
  },
  "subTableQueryConfigs": {
    "entry": {
      "enabled": true,
      "tableName": "t_sale_order_entry",
      "defaultConditions": [
        {
          "field": "order_id",
          "operator": "eq",
          "value": "${billNo}"
        }
      ]
    },
    "cost": {
      "enabled": true,
      "tableName": "t_sale_order_cost",
      "defaultConditions": [
        {
          "field": "order_id",
          "operator": "eq",
          "value": "${billNo}"
        }
      ]
    }
  }
}
```

### 前端调用

```javascript
import multiTableQueryBuilder from '../utils/multiTableQueryBuilder'

// 1. 解析配置
const subTableConfigs = multiTableQueryBuilder.parseSubTableConfigs(config)

// 2. 并行查询
const results = await multiTableQueryBuilder.queryAllSubTables(
  moduleCode,
  subTableConfigs,
  { billNo }
)

// 3. 使用结果
if (results.entry) {
  entryList.value = results.entry.data
}
if (results.cost) {
  costData.value = results.cost.data[0] || {}
}
```

## 🎯 最佳实践

###  DO

-  使用 `/erp/engine/query/execute` 接口
-  在 JSON 配置中定义 tableName 和 queryConfig
-  使用字段映射工具方法处理字典数据
-  为所有字典配置 TTL 缓存
-  使用 `${billNo}` 占位符传递上下文数据

### ❌ DON'T

- ❌ 为每个表创建单独的 Service
- ❌ 在 Controller 中硬编码 SQL
- ❌ 在配置中直接写 SQL 语句
- ❌ 忘记配置权限检查
- ❌ 忽略字段映射导致前端无法识别
