- `BusinessConfigurable.vue` 中调用了未定义的 `getOrderEntry()` 和 `getOrderCost()` 方法
- 会导致运行时错误

**修复方案**:

 **修改文件**: `BusinessConfigurable.vue`

**位置**:
```
d:\baiyuyunma\gitee-baiyu\baiyu-web\src\views\k3\pageTemplate\configurable\BusinessConfigurable.vue
```

**修改内容**:
- 第 855-861 行：将 `getOrderEntry()` 改为使用 `getApiMethod('entry')` 动态获取
- 第 869-878 行：将 `getOrderCost()` 改为使用 `getApiMethod('cost')` 动态获取

**修改前**:
```javascript
const entryResponse = await getOrderEntry(row.fbillNo)  //  未定义
const costResponse = await getOrderCost(row.fbillNo)    //  未定义
```

**修改后**:
```javascript
const entryMethod = await getApiMethod('entry')  //  动态获取
if (entryMethod) {
  const entryResponse = await entryMethod(row.fbillNo)
  // ...
}

const costMethod = await getApiMethod('cost')  //  动态获取
if (costMethod) {
  const costResponse = await costMethod(row.fbillNo)
  // ...
}
```

### **问题 3: 错误处理不统一**

**问题描述**:
- 部分方法返回 null，部分抛异常
- 响应码检查重复多次

**修复方案**:

 **创建文件**: `erpEngineHelper.js`

**位置**:
```
d:\baiyuyunma\gitee-baiyu\baiyu-web\src\utils\erpEngineHelper.js
```

**内容**:
```javascript
// 统一响应检查
export const isSuccess = (response) => {
  return response.code === 200 || response.code === 0 || response.errorCode === 0
}

// 统一响应处理
export const handleEngineResponse = (response, errorMessage = '操作失败') => {
  if (isSuccess(response)) {
    return response.data
  } else {
    throw new Error(response.msg || errorMessage)
  }
}

// 安全调用 (返回 null 不抛异常)
export const safeCallEngine = async (apiMethod, params) => {
  try {
    const response = await apiMethod(params)
    return isSuccess(response) ? response.data : null
  } catch (error) {
    console.error('引擎调用失败:', error)
    return null
  }
}
```
