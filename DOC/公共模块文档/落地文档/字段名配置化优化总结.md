#  字段名配置化优化完成总结

## 🎯 **优化目标**
实现前后端字段名**完全一致**，**无硬编码**，**全部从配置中动态读取**

---

##  **已完成的优化**

### 1. **Vue 组件优化** (`BusinessConfigurable.vue`)

#### 修复前（ 硬编码）：
```javascript
// 第 906-915 行
const billNoField = parsedConfig.page?.billNoField || 'fbillNo'
const actualBillNoField = billNoField.charAt(0).toUpperCase() + billNoField.slice(1)
const titleTemplate = businessConfig.value.drawerTitle || '{entityName}详情 - {fbillNo}'
drawerTitle.value = titleTemplate
  .replace(/{entityName}/g, entityName)
  .replace(/{fbillNo}/g, row[actualBillNoField] || '')
```

#### 修复后（ 完全配置化）：
```javascript
// 第 902-920 行
const billNoField = parsedConfig.page?.billNoField || 'FBillNo'
const titleTemplate = businessConfig.value.drawerTitle || '{entityName}详情 - {billNo}'
drawerTitle.value = titleTemplate
  .replace(/{entityName}/g, entityName)
  .replace(/{billNo}/g, row[billNoField] || row.FBillNo || '')
```

**关键改进：**
-  移除首字母大写的转换逻辑
-  模板变量从 `{fbillNo}` 改为 `{billNo}`（更通用）
-  直接使用配置的字段名访问数据
-  提供降级方案（`row.FBillNo`）

## 🎯 **核心原则**

### 1. **字段名直连映射**
```
数据库 → 配置文件 → Vue 组件
 FBillNo → FBillNo → row[FBillNo]
```
-  无转换
-  无计算
-  直接访问

### 2. **配置优先**
```javascript
// 优先级顺序
1. parsedConfig.page.billNoField（配置）
2. 'FBillNo'（默认值，与数据库一致）
3. row.FBillNo（降级方案）
```

### 3. **模板通用化**
```json
// 从特定字段名解耦
"{fbillNo}" → "{billNo}"
```
支持任意单据编号字段名，如：
- `FBillNo`（销售订单）
- `FBillNo`（采购订单）
- `FNoticeNo`（其他业务）

## 💡 **最佳实践**

### 1. **字段命名规范**
```
 推荐：FBillNo, FDate, FCustId（金蝶 K3 标准）
 避免：fbillNo, fdate, fcustId（小写格式）
```

### 2. **配置编写原则**
```json
//  正确：与数据库字段名完全一致
{
  "billNoField": "FBillNo"
}

//  错误：需要转换
{
  "billNoField": "fbillNo"  // 需要在代码中转换
}
```

### 3. **模板变量使用**
```json
//  推荐：通用变量名
{
  "drawerTitle": "{entityName}详情 - {billNo}"
}

//  避免：特定字段名
{
  "drawerTitle": "{entityName}详情 - {fbillNo}"
}
```
