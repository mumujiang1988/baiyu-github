## 🚀 一键部署脚本

### 方式一：使用 PowerShell 脚本 (推荐)

创建文件 `deploy-erp-fix.ps1`:

```powershell
# ============================================
# ERP 配置化方案修复部署脚本
# ============================================

Write-Host "========================================" -ForegroundColor Cyan
Write-Host "ERP 配置化方案修复部署" -ForegroundColor Cyan
Write-Host "========================================" -ForegroundColor Cyan
Write-Host ""

# 1. 检查环境
Write-Host "[1/5] 检查环境..." -ForegroundColor Yellow

# 检查 Java
$javaVersion = java -version 2>&1
if ($LASTEXITCODE -ne 0) {
    Write-Host "❌ Java 未安装，请先安装 Java 17+" -ForegroundColor Red
    exit 1
} else {
    Write-Host "✅ Java 已安装：$($javaVersion[0])" -ForegroundColor Green
}

# 检查 Maven
mvn -version | Out-Null
if ($LASTEXITCODE -ne 0) {
    Write-Host "❌ Maven 未安装" -ForegroundColor Red
    exit 1
} else {
    Write-Host "✅ Maven 已安装" -ForegroundColor Green
}

# 检查 Node.js
node -v | Out-Null
if ($LASTEXITCODE -ne 0) {
    Write-Host "❌ Node.js 未安装" -ForegroundColor Red
    exit 1
} else {
    Write-Host "✅ Node.js 已安装" -ForegroundColor Green
}

Write-Host ""

# 2. 编译后端
Write-Host "[2/5] 编译后端代码..." -ForegroundColor Yellow
Set-Location "d:\baiyuyunma\gitee-baiyu\baiyu-ruoyi"
mvn clean compile -DskipTests

if ($LASTEXITCODE -eq 0) {
    Write-Host "✅ 后端编译成功" -ForegroundColor Green
} else {
    Write-Host "❌ 后端编译失败" -ForegroundColor Red
    exit 1
}

Write-Host ""

# 3. 打包后端 (可选)
Write-Host "[3/5] 打包后端..." -ForegroundColor Yellow
mvn package -DskipTests

if ($LASTEXITCODE -eq 0) {
    Write-Host "✅ 后端打包成功" -ForegroundColor Green
} else {
    Write-Host "⚠️  后端打包失败 (但不影响开发模式运行)" -ForegroundColor Yellow
}

Write-Host ""

# 4. 安装前端依赖
Write-Host "[4/5] 安装前端依赖..." -ForegroundColor Yellow
Set-Location "d:\baiyuyunma\gitee-baiyu\baiyu-web"

if (Test-Path "node_modules") {
    Write-Host "⚠️  检测到 node_modules，跳过依赖安装" -ForegroundColor Yellow
} else {
    npm install
    
    if ($LASTEXITCODE -eq 0) {
        Write-Host "✅ 前端依赖安装成功" -ForegroundColor Green
    } else {
        Write-Host "❌ 前端依赖安装失败" -ForegroundColor Red
        exit 1
    }
}

Write-Host ""

# 5. 启动服务
Write-Host "[5/5] 启动服务..." -ForegroundColor Yellow
Write-Host ""
Write-Host "========================================" -ForegroundColor Cyan
Write-Host "部署完成！请执行以下命令启动服务：" -ForegroundColor Green
Write-Host "========================================" -ForegroundColor Cyan
Write-Host ""
Write-Host "1. 启动后端 (新窗口):" -ForegroundColor Yellow
Write-Host "   cd d:\baiyuyunma\gitee-baiyu\baiyu-ruoyi\ruoyi-admin-wms" -ForegroundColor White
Write-Host "   mvn spring-boot:run" -ForegroundColor White
Write-Host ""
Write-Host "2. 启动前端 (新窗口):" -ForegroundColor Yellow
Write-Host "   cd d:\baiyuyunma\gitee-baiyu\baiyu-web" -ForegroundColor White
Write-Host "   npm run dev" -ForegroundColor White
Write-Host ""
Write-Host "3. 访问系统:" -ForegroundColor Yellow
Write-Host "   http://localhost:8080" -ForegroundColor White
Write-Host ""
Write-Host "========================================" -ForegroundColor Cyan
```

**使用方法**:
```powershell
cd d:\baiyuyunma\gitee-baiyu
.\deploy-erp-fix.ps1
```

### 方式二：手动部署 (分步执行)

#### 步骤 1: 编译后端

```bash
# 进入后端目录
cd d:\baiyuyunma\gitee-baiyu\baiyu-ruoyi

# 清理并编译
mvn clean compile -DskipTests

# 预期输出:
# [INFO] BUILD SUCCESS
# [INFO] Total time:  XX seconds
```

#### 步骤 2: 打包后端 (可选)

```bash
# 打包 (用于生产部署)
mvn package -DskipTests

# 生成的 jar 包位置:
# baiyu-ruoyi/ruoyi-admin-wms/target/ruoyi-admin-wms.jar
```

#### 步骤 3: 检查前端依赖

```bash
# 进入前端目录
cd d:\baiyuyunma\gitee-baiyu\baiyu-web

# 如果 node_modules 不存在，安装依赖
if (!(Test-Path "node_modules")) {
    npm install
}
```

#### 步骤 4: 启动后端服务

**方式 A: 使用 Maven (开发模式)**
```bash
cd d:\baiyuyunma\gitee-baiyu\baiyu-ruoyi\ruoyi-admin-wms
mvn spring-boot:run
```

**方式 B: 直接运行 JAR (生产模式)**
```bash
cd d:\baiyuyunma\gitee-baiyu\baiyu-ruoyi\ruoyi-admin-wms
java -jar target/ruoyi-admin-wms.jar
```

**方式 C: 后台运行 (Windows)**
```bash
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd 'd:\baiyuyunma\gitee-baiyu\baiyu-ruoyi\ruoyi-admin-wms'; mvn spring-boot:run"
```

#### 步骤 5: 启动前端服务

**方式 A: 使用 npm**
```bash
cd d:\baiyuyunma\gitee-baiyu\baiyu-web
npm run dev
```

**方式 B: 后台运行 (Windows)**
```bash
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd 'd:\baiyuyunma\gitee-baiyu\baiyu-web'; npm run dev"
```

## 🔧 故障排查

### 问题 1: 后端启动失败

**错误**: `Port 8080 was already in use`

**解决方案**:
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080

# 终止进程 (替换 PID 为实际值)
taskkill /F /PID <PID>
```

### 问题 2: 前端编译失败

**错误**: `Module not found`

**解决方案**:
```bash
cd d:\baiyuyunma\gitee-baiyu\baiyu-web

# 删除 node_modules 和 package-lock.json
Remove-Item -Recurse -Force node_modules
Remove-Item -Force package-lock.json

# 重新安装依赖
npm install
```

### 问题 3: 数据库连接失败

**错误**: `Communications link failure`

**解决方案**:
1. 检查 MySQL 服务是否启动
2. 检查配置文件 `application.yml` 中的数据库连接信息
3. 确认数据库 `test` 存在
