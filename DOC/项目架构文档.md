# 项目架构文档

## 项目概述

### 1.1 项目背景

本项目是一个集供应链管理、客户管理、物料管理和数据分析于一体的企业级应用系统。系统采用前后端分离架构，前端基于Vue3 + Element Plus构建，后端基于Spring Boot + MyBatis Plus开发，集成金蝶K3 Cloud ERP系统，实现业务数据的无缝对接与管理。

- Vue 3.5.17
- TypeScript 5.8.3
- Element Plus 2.13.0
- Vite 构建工具

**后端技术栈：**
- Spring Boot 2.x
- MyBatis Plus
- MySQL / SQL Server 数据库
- 金蝶K3 Cloud SDK
- Redis 缓存
- Minio 对象存储

**开发工具：**
- Git 版本控制

- GitNexus 代码知识图谱

@Component
public class K3FormProcessorFactory {
    
    @Autowired
    private MaterialFormProcessor materialFormProcessor;
    @Autowired
    private SupplierFormProcessor supplierFormProcessor;
    @Autowired
    private PriceListFormProcessor priceListFormProcessor;
    @Autowired
    private InquiryOrderFormProcessor inquiryOrderFormProcessor;
    @Autowired
    private PurchaseQuotationFormProcessor purchaseQuotationFormProcessor;
    
    private final Map<String, AbstractK3FormProcessor<?>> processorMap = new HashMap<>();
    
    @PostConstruct
    public void init() {
        processorMap.put("BD_MATERIAL", materialFormProcessor);
        processorMap.put("BD_Supplier", supplierFormProcessor);
        processorMap.put("PUR_PriceCategory", priceListFormProcessor);
        processorMap.put("SAL_Inquiry", inquiryOrderFormProcessor);
        processorMap.put("PUR_Quotation", purchaseQuotationFormProcessor);
    }
    
    public <T> AbstractK3FormProcessor<T> getProcessor(String formId) {
        AbstractK3FormProcessor<?> processor = processorMap.get(formId);
        if (processor == null) {
            throw new IllegalArgumentException("不支持的表单类型: " + formId);
        }
        return (AbstractK3FormProcessor<T>) processor;
    }
}

### 2.4 抽象处理器

**模板方法模式：** [AbstractK3FormProcessor.java](file:///d:/baiyuyunma/By-middleground-web/ruoyi-admin-wms/src/main/java/com/ruoyi/business/k3/config/AbstractK3FormProcessor.java)

```java
public abstract class AbstractK3FormProcessor<T> {
    
    public Result processForm(MultipartFile[] files, T formData) {
        // 统一表单处理流程
        try {
            // 1. 数据验证
            validate(formData);
            
            // 2. 数据转换
            Map<String, Object> model = convertToModel(files, formData);
            
            // 3. 单据暂存
            String fid = draftForm(model);
            
            // 4. 附件上传
            uploadAttachments(files, fid, formData);
            
            // 5. 单据提交
            submitForm(fid);
            
            // 6. 单据审核
            auditForm(fid);
            
            return Result.success("处理成功");
        } catch (Exception e) {
            log.error("表单处理失败", e);
            return Result.error("处理失败: " + e.getMessage());
        }
    }
    
    // 抽象方法由子类实现
    protected abstract void validate(T formData);
    protected abstract Map<String, Object> convertToModel(MultipartFile[] files, T formData);
    protected abstract String draftForm(Map<String, Object> model);
    protected abstract void uploadAttachments(MultipartFile[] files, String fid, T formData);
    protected abstract void submitForm(String fid);
    protected abstract void auditForm(String fid);
}
```

public class MaterialFormProcessor extends AbstractK3FormProcessor<Bymaterial> {
    
    @Override
    protected void validate(Bymaterial formData) {
        if (StringUtils.isBlank(formData.getNumber())) {
            throw new IllegalArgumentException("物料编码不能为空");
        }
        if (StringUtils.isBlank(formData.getName())) {
            throw new IllegalArgumentException("物料名称不能为空");
        }
    }
    
    @Override
    protected Map<String, Object> convertToModel(MultipartFile[] files, Bymaterial formData) {
        Map<String, Object> model = new HashMap<>();
        model.put("FNumber", formData.getNumber());
        model.put("FName", formData.getName());
        model.put("FSpecification", formData.getSpecification());
        
        // 物料分组转换
        Map<String, Object> materialGroupMap = new HashMap<>();
        materialGroupMap.put("FNumber", getMaterialGroupCode(formData.getMaterialgroup()));
        model.put("FMaterialGroup", materialGroupMap);
        
        return model;
    }

import request from '@/utils/request'

// 获取客户分页列表
export function listCustomer(queryParams) {
  return request({
    url: '/k3/customer/list',
    method: 'get',
    params: queryParams
  })

// 新增客户
export function addCustomer(data) {
  const formData = new FormData()
  const { fKhlogoFile, ...customerData } = data
  
  const customerBlob = new Blob([JSON.stringify(data)], {type: 'application/json'})
  formData.append('customer', customerBlob)
  
  if (fKhlogoFile) {
    formData.append('logoFile', fKhlogoFile)
  }
  
  return request({
    url: '/k3/customer/save',
    method: 'post',
    data: formData,
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}

// 修改客户
export function updateCustomer(data) {
  const formData = new FormData()
  const { fKhlogoFile, ...customerData } = data
  
  const customerBlob = new Blob([JSON.stringify(data)], {type: 'application/json'})
  formData.append('customer', customerBlob)
  
  if (fKhlogoFile) {
    formData.append('logoFile', fKhlogoFile)
  }
  
  return request({
    url: '/k3/customer/update',
    method: 'post',
    data: formData,
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}

// 删除客户
export function deleteCustomer(idsToDelete) {
  const id = Array.isArray(idsToDelete) ? idsToDelete.join(',') : idsToDelete
  return request({
    url: `/k3/customer/delete/${id}`,
    method: 'delete'
  })
}

// 获取客户详情
export function getCustomer(id) {
  return request({
    url: '/k3/customer/query',
    method: 'get',
    params: { id }
  })
}

## 动态路由数据生成

### 5.1 路由系统架构

系统采用动态路由机制，菜单通过数据库配置动态生成，无需在路由文件中手动配置。

```
路由生成流程：
1. 用户登录 → 权限验证 → 获取用户菜单
2. 菜单数据转换为路由配置
3. 动态添加到Vue Router中
4. 路由守卫控制访问权限
```

### 5.2 动态路由原理

**路由配置存储：** 菜单信息存储在数据库`sys_menu`表中，包含以下字段：

### 5.3 动态路由生成过程

**路由生成核心代码：** [permission.js](file:///d:/baiyuyunma/By-web/src/store/modules/permission.js)

```javascript
import auth from '@/plugins/auth'
import router, { constantRoutes, dynamicRoutes } from '@/router'
import { getRouters } from '@/api/menu'
import Layout from '@/layout/index'
import ParentView from '@/components/ParentView'
import InnerLink from '@/layout/components/InnerLink'

// 匹配views里面所有的.vue文件
const modules = import.meta.glob('./../../views/**/*.vue')

const usePermissionStore = defineStore(
  'permission',
  {
    state: () => ({
      routes: [],
      addRoutes: [],
      defaultRoutes: [],
      topbarRouters: [],
      sidebarRouters: []
    }),
    actions: {
      setRoutes(routes) {
        this.addRoutes = routes
        this.routes = constantRoutes.concat(routes)
      },
      setDefaultRoutes(routes) {
        this.defaultRoutes = constantRoutes.concat(routes)
      },
      setTopbarRoutes(routes) {
        this.topbarRouters = routes
      },
      setSidebarRouters(routes) {
        this.sidebarRouters = routes
      },
      generateRoutes(roles) {
        return new Promise(resolve => {
          // 向后端请求路由数据
          getRouters().then(res => {
            const sdata = JSON.parse(JSON.stringify(res.data))
            const rdata = JSON.parse(JSON.stringify(res.data))
            const defaultData = JSON.parse(JSON.stringify(res.data))
            const sidebarRoutes = filterAsyncRouter(sdata)
            const rewriteRoutes = filterAsyncRouter(rdata, false, true)
            const defaultRoutes = filterAsyncRouter(defaultData)
            const asyncRoutes = filterDynamicRoutes(dynamicRoutes)
            asyncRoutes.forEach(route => { router.addRoute(route) })
            this.setRoutes(rewriteRoutes)
            this.setSidebarRouters(constantRoutes.concat(sidebarRoutes))
            this.setDefaultRoutes(sidebarRoutes)
            this.setTopbarRoutes(defaultRoutes)
            resolve(rewriteRoutes)
          })
        })
      }
    }
  })

// 遍历后台传来的路由字符串，转换为组件对象
function filterAsyncRouter(asyncRouterMap, lastRouter = false, type = false) {
  return asyncRouterMap.filter(route => {
    if (type && route.children) {
      route.children = filterChildren(route.children)
    }
    if (route.component) {
      // Layout ParentView 组件特殊处理
      if (route.component === 'Layout') {
        route.component = Layout
      } else if (route.component === 'ParentView') {
        route.component = ParentView
      } else if (route.component === 'InnerLink') {
        route.component = InnerLink
      } else {
        route.component = loadView(route.component)
      }
    }
    if (route.children != null && route.children && route.children.length) {
      route.children = filterAsyncRouter(route.children, route, type)
    } else {
      delete route['children']
      delete route['redirect']
    }
    return true
  })
}

function filterChildren(childrenMap, lastRouter = false) {
  var children = []
  childrenMap.forEach((el, index) => {
    if (el.children && el.children.length) {
      if (el.component === 'ParentView' && !lastRouter) {
        el.children.forEach(c => {
          c.path = el.path + '/' + c.path
          if (c.children && c.children.length) {
            children = children.concat(filterChildren(c.children, c))
            return
          }
          children.push(c)
        })
        return
      }
    }
    if (lastRouter) {
      el.path = lastRouter.path + '/' + el.path
    }
    children.push(el)
  })
  return children
}

// 动态加载组件
function loadView(component) {
  return modules[`/src/views/${component}`]
}

// 过滤动态路由
function filterDynamicRoutes(routes) {
  return routes.filter(route => {
    return route.hidden !== true
  })
}

export default usePermissionStore
```

### 7.2 常见问题

**问题1：菜单不显示**
- 检查数据库`sys_menu`表中的菜单配置
- 确认用户角色是否有菜单权限
- 检查路由配置是否正确

**问题2：API接口404错误**
- 检查前端API接口路径是否与后端Controller路径一致
- 确认后端服务是否正常启动
- 检查请求方法是否正确（GET/POST/PUT/DELETE）

**问题3：权限控制不生效**
- 检查数据库`sys_permission`表中的权限配置
- 确认用户角色是否有权限
- 检查v-hasPermi指令的权限标识是否正确
